## 一、战报数据编码格式（已完全破解）

```
原始字段 = base64( [4字节header(明文长度,小端)] + zlib( JSON文本 ) )

team_battle_data:  header=688 → 解压后688字节明文
enemy_battle_data: header=5172 → 解压后5172字节明文
```

**解码步骤**：`base64decode → 跳过前4字节 → zlib.decompress → UTF-8文本 → JSON/Python数组`

## 二、战报数据的完整结构

战报分两份：`team_battle_data`（我方视角）+ `enemy_battle_data`（敌方视角）。
以 enemy 为例，解压后是 JSON：`{"<team_id>": [块0, 块1, 块2]}`

### 块0：舰队配置
```python
[[2404, 51201, 7, 0, 0, 0, [172,0,0,78], "101,202,303,304,405,406,507,608,709",
  "5120107,1,9353,1,19,8601,1,20,8210,1;5120104,20,8207,1;...", ...]]
```
- `2404` = ship_id（舰船型号）
- `51201` = blueprint_id（蓝图）
- `7` = ship_type
- `[172,0,0,78]` = 坐标/等级相关
- `"101,202,..."` = **武器槽位列表**（对应slot_index）
- 长字符串 = **模块/科技配置**（格式 `module_id,param,...;`）

### ★ 块1：HP曲线（逐5秒采样，核心数据！）
```
格式: "type,time,struct_hp,shield_hp#type,time,struct_hp,shield_hp#..."
```
实测样本（敌方航母）：
```
5,0,370192,186208      ← t=0:   结构370192(满) 护盾186208(满)
5,5,370192,186208      ← t=5:   未掉血
5,10,370192,186208     ← t=10:  未掉血
5,15,370170,186108     ← t=15:  ★首次掉血!（首发延迟≈15s）
5,40,359892,184933     ← t=40:  持续下降
...
5,1080,1798,142623     ← t=1080(18分钟): 濒死
97,1082,7501766,0      ← t=1082: type97=击毁事件! uid=7501766(攻击方)
```

**关键发现**：
- **首次掉血在 t=15s** —— 印证04文档"首发延迟"（锁定+前摇）
- **总时长1082s** —— 与战报 end_time-begin_time = 1782891104-1782890021 = 1083s 吻合
- **type=5 是常规HP采样，type=97 是击毁事件**
- **采样间隔固定5秒** —— 不是逐发记录，是时间轴快照

**攻击事件格式**：`[event_type, [[attacker_uid, val1, val2]...]]`

| event_type | 含义（推断） | val1 | val2 |

| 2 | 持续伤害/输出 | begin_time | 36280=总伤害? |
| **5** | **击毁事件** | begin_time | **end_time(击毁时刻)** |
| 8 | 另一类伤害统计 | begin_time | 36280 |

## 四、对战报数据价值的评估

### ✅ 已获得的能力
1. **读取任意战报的完整HP曲线**（逐5秒的结构值+护盾值）
2. **读取攻击事件**（每艘舰对每个目标的伤害统计+击毁时刻）
3. **读取舰队配置**（ship_id/blueprint/武器槽/模块科技）
4. **读取舰船基础数值**（结构值/护盾值）

### ⚠️ 局限
1. **HP曲线是5秒采样**，不是逐发记录 → 看不到单发命中/未命中
2. **攻击事件是汇总级**（总伤害36280），不是逐发damage_list
3. **事件类型0/2/8的精确含义待更多样本确认**
4. 需要**手动点开战报**才能加载（非自动批量）

### 对04文档核心问题的帮助
- **命中率判定形式**：HP曲线5秒采样无法直接看逐发命中，但**攻击事件的总伤害**可用于校验
- **+30%命中加成衰减**：如果有+0%/+30%两档的战报，可对比总伤害比值（比黑盒时长反推更准）
- **确定性验证**：同配置重跑两场，对比HP曲线是否完全相同 → 直接验证NV1可复现性
