> **本组数据专门验证两条主公式的"破甲"分支**，与第一组（保底分支）互补，
> 两条分支被独立采样验证。**两条核心公式经两组实测交叉验证，已可确信。**

| 项目 | 公式 | 实测反推 / 蒙特卡洛 | 引擎状态 |
|------|------|---------------------|---------|
| **抵抗减伤主公式（破甲分支）** | `实际单发 = dph − resistance` | 105−8=**97**，引擎=97 | ✅ 与实测完全一致 |
| **命中/闪避公式** | `最终命中 = base × (1 − dodge)` | 0.6×(1−0.08)=**55.2%**，MC收敛55.25% | ✅ 误差0.05pp |
| **总伤害量级** | 146命中×97≈14162 | 蒙特卡洛平均 **14063** | ✅ 偏差仅 −1.41% |

### 实测数据
```
攻击方：驱逐舰，5 个相同主武器
  单武器：dph=105、冷却6s、无攻击持续时间（单发/循环）、锁定3s、命中率50-70%（面板区间）
防守方：FG300（抵抗8、闪避8%、结构值14263）
结果：FG300 在 5分22秒（322s）被击毁，总伤害 = 14263（=结构值）
```

### 命中率区间建模（面板特性）
游戏面板命中率显示为**区间**（如 50%-70%），引擎把 `baseHit` 存为 `{min:0.5, max:0.7}`，
**每发开火在区间内独立 roll** 一个 base 值，再套 `base×(1−dodge)` 公式做命中判定。
区间中值 0.6 用于纯函数估算（`computeHitRate`）。本组数据已用真实区间回归。

### 手工推演与引擎吻合情况
```
单发伤害 = 105 − 8 = 97           （破甲减法，dph>resistance）
命中率   = 区间中值 0.6 × (1 − 0.08) = 55.2%
每武器循环 = 6s/发，(322−3)/6 ≈ 53 发/武器，5 武器 = 265 发
命中发数 ≈ 265 × 0.552 = 146 发
总伤害   ≈ 146 × 97 = 14162（≈实测 14263，偏差0.7%）
```

回归测试见 `packages/engine/tests/dd-vs-fg300-real.test.ts`（6 个断言）。

## 0. 先明确我们要测什么

实弹伤害公式（用户提供）：

```
实际单发伤害 = 面板单发伤害(dph) − 抵抗值(resistance)    当 dph > resistance
实际单发伤害 = 保底伤害(floor)                          当 dph ≤ resistance
```

**两个待标定未知量：**

| 未知量 | 当前占位 | 说明 |
|--------|---------|------|
| `floor`（保底伤害） | 1 | 破不了甲时每发的最低伤害。最关键，优先测。 |
| 公式边界 | 假设 dph>res 才不保底 | 需验证：是严格 `>` 还是 `≥`？是否有其他修正？ |

**还要顺带验证的已知量**（用于交叉校验整个公式链）：
- 命中是否影响曲线（未命中秒 → 该秒伤害为0）
- 暴击是否 = ×2（暴击秒伤害是否正好翻倍）

## 1. 核心思路：用"逐秒伤害曲线"反推

逐秒伤害曲线给出的是 `某秒内该武器对该目标造成的总伤害`。

```
单秒总伤害 = (该秒命中发数) × 单发实际伤害
```

只要我们**控制变量**让"单发实际伤害"是常数，就能从曲线反推：

```
单发实际伤害 = 单秒总伤害 ÷ 该秒命中发数
```

所以测试的本质是：**构造已知 dph、已知 resistance、已知命中发数的场景，读曲线算单发伤害，对比公式。**

---

## 2. 测试一：标定保底伤害 floor（最优先）

### 目的
测出"破不了甲"时每发到底打多少。

### 场景设计
构造 **dph ≤ resistance** 的攻击：
- 攻击方：选一艘**单发伤害极低**的舰船/武器（如某些护航艇的小口径炮、或低强化武器，dph 越小越好，理想 dph ≤ 10）
- 防守方：选一艘**抵抗值极高**的舰船（如战列舰/战巡，resistance 可达 30~60+）
- 确保 `dph < resistance` 成立

### 操作步骤
1. 记录攻击武器的**面板单发伤害 dph**（游戏内武器详情页）
2. 记录防守舰船的**抵抗值 resistance**（装甲系统详情页）
3. 打一场战斗，**只有这一个武器在打这个目标**（避免其他武器干扰曲线）
   - 实操：用 1v1，攻击方只带这 1 个对舰武器；防守方不带能反击的同排武器，或选不会反击的目标
4. 战后打开**逐秒伤害曲线**，筛选出该武器对该目标的伤害

### 数据读取与计算
对曲线中**每个有伤害的秒**：
```
单发实际伤害 = 该秒总伤害 ÷ 该秒命中发数
```
- 该秒命中发数：从武器攻击循环算（持续期间每秒打几发）。若不确定，**取多个伤害秒的值取众数/最小值**——因为保底是固定的，所有非暴击秒的单发值应该完全相等。
- **排除暴击秒**（伤害明显翻倍的秒）。

### 预期结果
所有非暴击秒算出的"单发实际伤害"应该是一个**固定常数**，这个常数就是 `floor`。

> ⚠️ 如果不同秒算出的值有波动（不是固定常数），说明保底不是固定值，可能有随机成分——那就要记录波动范围，后续按分布建模。

## 3. 测试二：验证减法主公式 `dph − resistance`

### 目的
确认主公式就是简单减法（而非某种系数/分段）。

### 场景设计
构造 **dph 明显 > resistance** 的攻击，且 resistance ≠ 0：
- 攻击方：单发伤害 dph 适中（如 100~300）
- 防守方：抵抗值 resistance 已知（如 10、20）
- 同样保证 1v1、单武器打单目标

### 数据读取
同测试一，从逐秒曲线反推单发实际伤害。

### 验证
```
反推单发伤害 是否等于 (dph − resistance)？
```
- **相等** → 减法公式确认，主公式搞定
- **不等** → 公式有额外修正（如最低保留比例、四舍五入、额外减伤词条），需进一步排查

### 进阶：多点采样画曲线
固定 dph，换不同 resistance 的防守舰（resistance = 5/10/15/20/30...），各测一次单发伤害，画出 `单发伤害 vs resistance` 散点图：
- 若是纯减法 → 一条斜率 −1 的直线，截距 = dph
- 若有拐点/分段 → 直线在某处变平（那个平的值就是 floor 的边界点）

这一步能**同时验证主公式和 floor 的触发边界**。

---

## 4. 测试三：交叉校验（命中 / 暴击）

### 4a. 命中
- 看逐秒曲线：**未命中的秒，伤害应为 0**（曲线在该秒无数据点或为0）
- 统计"有伤害的秒数 / 总开火秒数" → 反推命中率，对比面板基础命中 ± 闪避
- 这能验证命中公式链是否正确

### 4b. 暴击倍率
- 找曲线中**伤害正好是普通秒 2 倍**的秒 → 确认暴击 = ×2
- 若不是正好 2 倍 → 测实际倍率（如 1.5、2.5）
- 统计暴击秒占比 → 反推暴击率，对比面板

## 5. 测试四：能量伤害公式（顺便测）

能量伤害公式（待验证）：`实际 = dph × (1 − shield)`

- 攻击方：能量武器（脉冲/离子炮/等离子）
- 防守方：已知护盾值 shield 的舰船
- 从曲线反推单发伤害，验证是否 = `dph × (1 − shield)`
- **同时验证能量伤害是否真的忽略抵抗**（防守方抵抗值高低不影响能量伤害）

## 7. 如何把结果喂给模拟器

采样完成后，改 `packages/engine/src/core/config.ts` 一处即可：

```typescript
export const DEFAULT_CONFIG: EngineConfig = {
  kineticFloorDamage: <实测保底值>,  // 例如测出来是 5 就填 5
};
```

然后跑 `npm run test:dump`，把模拟器的逐秒输出和游戏内曲线**逐秒对比**：
- 完全吻合 → 公式链全部校准成功
- 有偏差 → 定位是哪一秒开始偏、偏多少，反推是哪个环节（命中？暴击？循环节奏？）

## 9. 关键注意事项

1. **单变量原则**：每场测试只让一个武器打一个目标，否则曲线混在一起无法反推单发值。
2. **排除干扰**：防守方最好不反击（或反击打不到攻击方），避免攻击方中途死亡导致数据不全。
3. **足够样本**：逐秒曲线多取几个伤害秒（至少 5~10 个），取众数避免偶然。
4. **注意攻击循环节奏**：确认武器是"持续 N 秒打 M 发"还是"每秒打 X 发"，这决定"该秒命中发数"怎么算。若拿不准，**优先用伤害值出现重复的秒**（重复值 = 单发伤害或其整数倍，最易识别）。
5. **记录原始数据**：截图保留，万一后续发现公式有隐藏机制，原始数据是排查依据。
