## 1. 已证实的完整命中率公式

### 1.1 公式

```
final = base × ( 1 + hitBonus + hitBonusByTargetClass + buff加成
                    − dodge − dodgeByWeaponType − buff闪避 )
        再夹到 [10%, 95%]
```

**所有命中/闪避类词条都是同一对括号里的加法项**，区别只在符号(±)和作用域(全局/舰种/武器类别/临时)：

| 词条类型 | 槽 | 符号 | 作用域 | 引擎字段 |
|---|---|---|---|---|
| 通用闪避 | − | − | 全局 | `Ship.dodge` |
| 被直射/制导/慢速武器命中率下降 | − | − | 武器类别 | `Ship.dodgeByWeaponType` |
| 临时闪避buff（策略类） | − | − | 时效性 | `ActiveBuffs.dodge` |
| 蓝图强化命中 | + | + | 全局 | `WeaponSystem.hitBonus` |
| 对某舰种命中率提升 | + | + | 舰种 | `WeaponSystem.hitBonusByTargetClass` |
| 全局命中下降（策略类） | + | − | 全局/时效 | `ActiveBuffs.hitBonus` |

### 1.2 四组单舰实验铁证（均在引擎中有回归测试）

所有实验攻击方均为 **FG300 反击**（base 反推 ≈ 0.687），靶为 **卡利莱恩特种型**（护卫舰，闪避 55%，抵抗 8）。

**实验0 — 通用闪避**：反推 base 一致 ✓
**实验1 — 被直射命中率下降 k**（k=15%/30%）：
  - 加法预测 20.6% / 10.3%（触底），实测 20.8% / 10.1% ✓
  - 比值判据 0.486，贴加法 0.500，远离乘法 0.824
**实验2 — 对护卫舰命中率提升 k**（k=15%/30%，击杀时间法）：
  - 加法预测 41.2% / 51.5%，实测 40.4% / 50.0% ✓
  - 比值判据 1.230，贴加法 1.250，远离乘法 1.130
**实验3 — 全局命中下降15%**（阋神星武器2，弹药2=每次2发）：
  - 加法预测 20.6%，实测 20.94% ✓
  - 反推 base = 0.698，贴近标定 0.687

**三组实验从不同路径反推的 base 全部落在 0.67-0.70** → 公式从头到尾就是这一套，无隐藏项。

## 2. 临时 buff（策略类技能）实现

### 2.1 结论

临时 buff 修饰值**作为加法项叠加进命中括号**，与面板层修饰共用同一对槽，不走独立乘法通道。

**证据**：10艘斗牛跨度判据 T(+0%)−T(+30%) = 22.5s，加法预测 21.1s，乘法预测 13.8s → 贴加法。

**诚实标注**：此结论受多舰批次不同步干扰（见第3节），不如单舰实验干净。但加法方向明确（跨度差 12.8s 远超散布），且与"同一括号加法"范式一致。

### 2.2 引擎实现

- 类型：`TempBuff`（trigger: periodic/threshold + duration + stat + value）
- 运行时：`RuntimeShip` 跟踪 activeBuffs（含到期时刻）+ threshold 已触发集合
- 调度：`simulator` 用 `skillReady`/`skillExpire` 事件处理 periodic；threshold 在每次伤害结算后检查
- 命中判定：`hitCheck` 前收集 `ActiveBuffs` 传入
- 测试：`tests/tempbuff.test.ts`（8个）、`tests/tempbuff-scheduler.test.ts`（4个）

### 2.3 关键认知：临时buff"无敌"是设计意图

卡利莱恩 55%+30%直射+90%策略 = 175% → 撞10%下限，近乎无敌。**这不是 bug**：
- 乘法模型下也撞底（90%×0.10 把命中率压到 2%）
- 大数值 buff 本就为制造"无敌窗口"（保命技能）
- 引擎 10% 下限保护正确

## 3. ⚠️ 多舰调度问题（待解决，优先级高）

### 3.1 问题现象

用 10 艘满面板斗牛集火测试临时 buff 时，发现**模拟时长系统性偏短**（实测 74.5s vs 模拟 61.8s，差 13s）。深入分析后发现：

- **单舰实验全部精确吻合**（实验0-3）
- **唯一的多舰实验就出偏差**
- 10艘斗牛个体主武器命中发数 **5~10发（差2倍）**，但全部打满全场、无阵亡、无暴击
- 用户观察到"各斗牛发射先后不同"、"首次伤害在5s后（锁定是3-4s）"

### 3.2 初步结论：多舰存在【批次不同步】

证据链全部指向同一机制：**多艘船的开火不是同步的，存在系统性相位错开**。

可能原因：
1. 开火相位错开（避免齐射尖峰/平衡DPS）
2. 服务器批次处理（同tick开火事件排队）
3. 锁定时刻的 RNG 差异

### 3.3 已锁定的硬数据（无策略靶校准）

用一场 43s 无策略靶战斗（10艘斗牛，每艘独立伤害统计）反推：

| 项目 | 值 | 状态 |
|---|---|---|
| 主武器 perHit | 514（525×0.979） | ✅ 锁定（护盾3%减伤） |
| 副武器 perHit | 27（28×0.964） | ✅ 锁定 |
| 总伤害对账 | 51623 ≈ 血量51632 | ✅ 完美吻合 |
| "数量X"语义 | X门独立武器（非齐射） | ✅ 确认 |
| 首发延迟 | ~5s（锁定3-4s + 额外1-2s） | ✅ 用户观察 |

**未解决**：
- 后续发间间隔（反推互相矛盾：2.7s/5.0s/6.7s 都对不上）
- 锁定是仅首次还是每发
- 主武器发数为何波动2倍（相位错开的具体规律）

### 3.4 下一步计划

**用户回家后会提供 9 场 10斗牛单场的每艘个体伤害数据**（主武器/副武器）。需要分析：
1. 个体发数分布 → 推断相位错开规律（固定差？随机？排队？）
2. 发数与时间的关系 → 锁定真实发间间隔
3. 批次不同步的数学模型 → 写进引擎调度器

**这是后续多目标锁定/集火测试的前提**——不解决多舰调度，多目标场景的 DPS 计算都会失真。

## 4. 武器循环参数语义（已校正）

### 4.1 shotsPerCycle = 弹药 × 次数

- "弹药" = 每次开火打出的弹数（弹药2 = 一次打2发）
- "次数" = 一个循环内开火轮数
- 例：FG300武器1 "弹药1次数3" → shotsPerCycle=3；阋神星武器2 "弹药2次数1" → shotsPerCycle=2

### 4.2 ⚠️ "锁定时间"语义存疑

引擎当前假设"锁定仅在战斗开始首次生效"（`WeaponSystem.lockOnTime` 文档）。但实测显示：
- 首次伤害在5s后（锁定4s + 额外延迟）
- 主武器发数反推的周期（5.0s）既非"仅首锁+冷却2.7s"也非"每发锁+冷却6.7s"

**待多舰数据厘清后重新校正**。当前引擎的锁定模型可能需要修正。
