## 0. 本轮工作总览

本轮围绕"**10艘斗牛打靶，9场逐元素伤害完全相同**"这一反常现象展开，目标是搞清游戏的确定性机制，并让引擎的总时长对得上实测。

结论分三部分：

| 结论 | 状态 | 证据强度 |
|---|---|---|
| 伤害计算 = 期望伤害模式（N≥2，不 roll） | ✅ 锁定 | 多场逐元素相同 + 1V1随机 对照 |
| 随机源 = 跨排目标切换（非伤害计算） | ✅ 锁定 | 2V2排位对照实验 |
| 战报时长结构 = DES击毁 + 后摇5s | ✅ 锁定 | 战报坐标点位 + 反推起点偏移≈0 |
| 多舰 base = 区间下限 | ✅ 较强 | 无策略+0%档精确吻合 + 物理一致性 |
| **+30%命中加成在多舰下只兑现58%** | ❌ **新谜题** | 比值判据（与base无关）|

⚠️ **重要纠偏**：03文档的"多舰批次不同步"假说（§3）经审查**无工程必要性、无直接证据，予以放弃**。它是为填补偏差而虚构的机制。文档§3的相关内容不再作为指导，但保留历史记录。

## 1. 已锁定的机制（可写进引擎）

### 1.1 期望伤害模式（N≥2）

**结论**：攻击方舰船数 ≥ 2 时，伤害计算走"期望伤害"——每发扣 `perHit × finalHitRate`，不 roll 命中/未命中。N=1 时走原 per-shot roll。

**证据链**（实测对照）：

| 场景 | N | 攻击方受损? | 实测 | 解释 |
|---|---|---|---|---|
| 10打1 哑靶 | 10 | 否 | 逐元素完全相同（9场） | 期望伤害，无RNG |
| 10打1 无策略 | 10 | 否 | 3场完全相同 | 同上 |
| 10打2 同舰种 | 10 | 否 | 完全相同 | 同上 |
| 10打2 异舰种同排 | 10 | 否 | 完全相同 | 同上 |
| 2打2 同舰种 | 2 | — | 完全相同 | N=2已进入期望区 |
| 2打2 异舰种同排 | 2 | — | 完全相同 | 同上 |
| 1对1 | 1 | 是(互伤) | 每场不同 | N=1走per-shot roll |

**固化阈值 = 2**。N≥2 即期望伤害，N=1 才 roll。

**工程合理性**：期望伤害是 O(1) 算术、零RNG、跨平台确定。大舰队切期望模式可降服务器负载1-2个数量级，且保证PvP结算可复现、无争议。这是手游MMO的合理设计。

### 1.2 随机源 = 跨排目标切换（非伤害计算）

**结论**：游戏的随机性不在伤害计算，而在"目标选择"。具体来说，**跨排的目标切换是唯一随机源**。

**决定性证据**（2V2排位对照实验）：

| 2V2配置 | 排位 | 结果 |
|---|---|---|
| 异舰种 一前一中（不同排） | 跨排选择 | 随机 |
| 异舰种 同排 | 无跨排 | 完全相同 |

差异只在"排位布局"这一个变量。同排（无论同/异舰种）完全确定；不同排才随机。

**机制图景**：
- 同排内目标选择有确定性优先级（舰种/id）→ 无随机
- 跨排目标获取涉及"重新锁定/转火"，时刻分叉 → 随机源
- 伤害计算本身（期望伤害）从来不是随机源

**对单靶场景的含义**：单靶无目标选择 → 完全确定。4个时长锚点全是单靶，不受随机源影响，可直接校准。

### 1.3 战报时长结构

**结论**：`战报时长 = DES击毁时刻 + 后摇5s`。起点 ≈ 锁定开始时刻（不含观看延迟）。

**证据**：
- 后摇5s：战报坐标上有明确点位（非看曲线形状），击毁后5s空白
- 起点≈0：用下限base反推，4锚点的"起点偏移"全部在 −1.5~+3.3s（接近0）
- **动画时间 ≠ 战报时间**：用户进战场观看的延迟（>4s）是客户端渲染，不计入战报时长

⚠️ **"6s切线"不可信**：伤害曲线做了滑动平均平滑，切线位置 = 真实首炮 ± 平滑窗口/2，无法精确定位前摇。不能用切线推断前摇。

### 1.4 多舰 base = 区间下限

**结论**：多舰期望模式下，命中率公式的 base 取**区间下限值**（主0.70/副0.50），不是中值（0.85/0.60）。

**证据**：无策略+0%档（最干净）反推：
- 实测整场命中率：53.4%
- 下限base预测：53.3% ✓ 几乎完美
- 中值base预测：64.6% ✗ 偏高

**物理一致性判定**：下限base让4锚点的"起点偏移"接近常数（≈0）；中值base的起点偏移随k/buff乱跳（+6~+11），物理上说不通（前摇不该随命中率变）。下限base胜出。

⚠️ 这是用户提出的假设，被数据验证。但注意：这与1V1（per-shot roll，base≈0.67区间roll均值）是两套机制，量级一致但实现不同。

### 1.5 buff机制 + 加法公式（结构正确）

**结论**：buff触发、加法公式的**结构**完全正确。

**证据**（差值判据，与base/后摇无关）：
- 有策略靶 T(+0%)−T(+30%)：实测23s = 模拟23.3s ✓ 几乎完美
- 这个差值抵消了前摇/后摇/base，纯粹反映"buff+加法公式"的作用，完美吻合

**含义**：60%血触发+40%闪避/45s 的buff模型、加法叠加方式 都是对的。

## 2. ⚠️ 新谜题：+30%命中加成在多舰下衰减

### 2.1 现象

用最干净的**比值判据**（与base/后摇/DPS全部无关）：

```
无策略靶(闪避20%) 净时长(扣后摇5s):
  +0%命中:  45s
  +30%命中: 37s
  比值 T(+30%)/T(+0%) = 0.822
```

不同公式形式的预测（都与base无关，相除约掉）：

| 公式 | 预测比值 | 实测 | 吻合? |
|---|---|---|---|
| 加法 base×(1+k−d) | 0.727 | 0.822 | ❌ |
| 乘法 base×(1+k)×(1−d) | 0.769 | 0.822 | ❌ |

**两个标准公式都对不上。** 实测0.822比预测都大 → +30%命中加成的实际效果被压缩。

反推：+30%命中只兑现了 **58%** 的效果（衰减系数α=0.577）。

### 2.2 与单舰铁证的冲突

```
单舰(1V1, per-shot roll): +30%命中, 加法公式完美吻合(比值1.667=预测1.667) ✓
多舰(10V1, 期望伤害):     +30%命中, 加法公式失效(比值0.822≠预测0.727)    ✗
```

**同样的"+30%命中"，单舰全额生效，多舰只兑现58%。** 这是本轮最核心的未解问题。

### 2.3 已排除的解释

- ❌ **95%命中率上限夹取**：主武器+30%命中=0.77，不撞95%
- ❌ **base不同**：比值判据与base无关（相除约掉）；且主武器+30%不撞上限，两档base应相同
- ❌ **开火时序不同步（03文档§3假说）**：无工程必要性、无直接证据，已放弃
- ❌ **后摇/前摇**：比值判据已抵消

### 2.4 待验证的可能解释

1. 多舰模式下，命中加成k有独立的衰减机制（单舰不衰减）
2. "+30%命中提升"在多舰下走另一套算法（如加到base而非括号）
3. 存在未建模的机制（协同/饱和攻击类）

**无法从现有数据区分，需要专项实验（见§4）。**

## 3. 校准现状（4锚点）

### 3.1 已坐实的模型参数

```
伤害模式: 期望伤害(N≥2), 每发扣 perHit×finalHitRate
命中率:  加法 base×(1+k-dodge-buff闪避), 夹[10%,95%]
base:    区间下限(主0.70/副0.50)
后摇:    5s
perHit:  面板×(1-护盾) = 主509.25/副27.16
```

### 3.2 4锚点对照（下限base + 后摇5s）

| 配置 | 实测 | 模拟 | 误差 |
|---|---|---|---|
| 无策略+0% | 50s | 50.0s | 0.0 ✓ |
| 有策略+0% | 75s | 76.5s | +1.5 ✓ |
| 无策略+30% | 42s | 38.7s | −3.3 |
| 有策略+30% | 52s | 49.5s | −2.5 |

**+0%两档几乎完美，+30%两档系统性偏快2.5-3.3s** —— 正是§2的命中加成衰减。

## 4. 下一步实验方案

### 4.1 ★决定性实验：命中加成是否随舰船数量变化（用户建议）★

**目标**：判定+30%命中加成的衰减是"随N连续变化"还是"N≥2固定衰减"还是"N=1不衰减"。

**配置**：固定靶（闪避20%无策略），攻击方分别用 **1/2/3/4/5/10 艘斗牛**，每配置测 +0% 和 +30% 两档。

**记录**：每档战斗时长（5s后摇已知，扣掉得净时长）。

**判据**：算每个N的 `T(+30%)/T(+0%)` 比值：

| 观测到的比值模式 | 结论 |
|---|---|
| 所有N的比值都=0.727（加法预测） | 加法处处成立，多舰衰减是别的机制 |
| N=1比值=0.727，N≥2比值=0.822（跳变） | N=1和N≥2用不同命中率规则 |
| 比值随N从0.727渐变到0.822 | 衰减是连续曲线，需建模N的函数 |

**这个实验能直接判定§2谜题的机制类型，是最有价值的下一步。**

### 4.2 辅助实验：dodge=0靶的比值

**配置**：找零闪避靶（或抵抗极高的小船），10艘斗牛打，+0%/+30%两档。

dodge=0时比值判据更干净（无闪避项干扰）：
- 加法：T(+30%)/T(+0%) = 1/(1+k) = 0.769
- 若实测≠0.769，进一步确认加成衰减独立于闪避

### 4.3 待验证项（用户已部分测试）

- [ ] 10V2 不同排 是否随机（预期随机，坐实"跨排是唯一随机源"）
- [ ] 攻击方减员到1艘后，剩余伤害是否切回per-shot roll（2V2双方同舰种，看被击毁方分布）

## 5. 方法论纠偏记录（避免重蹈覆辙）

本轮调试过程中出现的过拟合/虚构机制错误，记录以防复发：

| 错误 | 表现 | 纠正 |
|---|---|---|
| 过拟合系数 | 0.75全局系数、C=15.3s前摇、0.82时序衰减 | 用户两次点破"又是过拟合吧"；放弃，回到差值/比值判据 |
| 虚构机制 | "开火时序不同步"(03文档§3) | 无工程必要、无直接证据；放弃 |
| 误信平滑曲线 | 用"6s切线"当前摇 | 平滑曲线切线不可靠，用户指出 |
| 时长反推歧义 | 用总时长反推base/DPS/公式，三者纠缠 | 改用比值判据（相除约掉未知量） |

**核心教训**：当多个未知量纠缠时，必须用**比值/差值判据**抵消未知量，而非反推单一系数。直接机制证据 > 间接时长反推。
