# 06-客户端白盒数据说明

> 日期：2026-07-01
> 数据来源：网易 NeoX 引擎客户端运行时注入 dump（frida + PyRun_SimpleString）
> 获取方法：见 `data/client/analysis/HOOK_REVIEW.md`（注入闭环）+ `BATTLE_REPORT_DECODED.md`（战报解码）

## 二、战报数据（battle_report/）— 服务器计算结果

### 2.1 编码格式

```
原始字段 = base64( [4字节长度header(小端)] + zlib( JSON/Python数组明文 ) )
解码: base64decode → 跳过前4字节 → zlib.decompress → UTF-8
```

### 2.2 数据结构（以 enemy_battle_data 为例）

解压后是 `{"<team_id>": [块0, 块1, 块2]}`

**块0**：舰队配置 `[ship_id, blueprint_id, ship_type, ..., 武器槽位, 模块科技串]`

**★ 块1**：HP曲线（逐5秒采样）
```
"type,time,struct_hp,shield_hp#..." 
type=5: 常规HP采样
type=97: 击毁事件
```

**★ 块2**：舰船详细 + 攻击事件
```
[玩家信息, [舰船记录], [攻击事件]]
攻击事件: [target_uid, weapon_id, [], [[event_type, [[attacker, val1, val2]]...]]]
  event_type=0: 命中/伤害
  event_type=2: 持续输出统计  
  event_type=5: ★击毁时刻(val2=end_time)
  event_type=8: 伤害统计
```

详见 `analysis/BATTLE_REPORT_DECODED.md`。

### 2.3 文件清单

| 文件 | 内容 |
|---|---|
| battle_data_structured.json | 结构化JSON（我方+敌方） |
| enemy_battle_data_decoded.txt | 敌方明文（含HP曲线，最有价值） |
| team_battle_data_decoded.txt | 我方明文 |
| *_decoded.bin | 二进制版 |
| *.txt (无decoded) | 原始base64编码 |
| report_summary.json | 战报摘要 |

## 三、模块结构（modules/）— 客户端类定义

从游戏Python解释器dump的类/方法/常量（字节码有自定义opcode，但元数据可读）。

| 文件 | 内容 |
|---|---|
| 01_battle_command.txt | ★31种战斗命令类（WeaponAttackCommand等） |
| 02_battle_entity.txt | 战斗实体/船组数据 |
| 03_ship_attribute.txt | 舰船属性类（最大，2576行） |
| 04_weapon.txt | 武器/子弹系统（Bullet类等） |
| 05_buff_effect.txt | buff效果框架 |
| 06_weapon_action_probe.txt | 武器动作字段探测 |
| **07_protocol_mapping.txt** | ★命令ID→命令类映射表 + BattleDamageType枚举 |

**07的关键内容**（引擎协议对齐用）：
```
BattleDamageType.HIT=1, MISS=-1, ANTI=-2
cmd_id=2: WeaponAttackCommand (武器攻击)
cmd_id=3: ShipDeadCommand (击毁)
cmd_id=13: ShipSystemDestroyCommand (子系统摧毁)
cmd_id=29: ShipHPUpdateCommand (HP更新)
```

## 四、对引擎的具体用法

### 4.1 数据加载（建议）

引擎可写一个 `clientData` 模块，启动时加载 config/ 下的JSON：

```typescript
import ship from '../../data/client/config/cfg_ship.json';
import weapon from '../../data/client/config/cfg_weapon.json';
// ship[ship_id][4] = 结构值
// weapon[weapon_id].ATTACK_INTERVAL = 攻击间隔
```

### 4.2 校准基准

战报HP曲线是**服务器原始数据**，可用于引擎输出校准：
- 引擎模拟一场战斗 → 输出逐5秒HP → 与战报HP曲线对比
- 不再是只能看总时长的黑盒

### 4.3 机制实现清单

module_effect 的 EFFECT_ID 列表 = 引擎要实现的机制TODO：
- 12156/12010: 命中率（已有加法公式，04文档）
- 12300: 单发伤害
- 12030/12031: 暴击
- 10010: 闪避
- 12080: 反导拦截
- 12370: 转火

### 4.4 字段映射待办

部分配置表的list型记录（舰船/蓝图/武器动作）字段索引含义未完全确认，需要：
1. 对照更多已知舰船数据反推
2. 或从 `modules/03_ship_attribute.txt` 的 `FIELDS` 定义查证

## 五、数据获取的可复现性

这套数据可通过 `analysis/HOOK_REVIEW.md` 的方法重新获取（游戏更新后）：

1. `pip install frida frida-tools`
2. 启动游戏，管理员运行 `inject_probe3.py` 验证通道
3. `batch_dump3.py` + `_dump_logic.py` dump模块结构
4. 战报：游戏内点开战报详情 → `read_battle_data.py` 读取解码
5. 配置表：`dump_all_blueprint.py` 批量导出

工具脚本保留在 `E:\星际猎人\`（游戏客户端目录）。
