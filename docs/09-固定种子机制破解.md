# 09-固定种子机制破解：种子 = 舰队标识符

> 日期：2026-07-02
> 性质：**决定性结论** —— 困扰多轮研究的"固定种子现象"真相大白
> 前置：04-多舰确定性与期望伤害模式交接.md（其核心未解问题在此收口）
> 方法论：关键变量识别（舰队ID）+ 控制变量对照实验

---

## 〇、一句话结论

**服务器在战斗开始时，用"舰队标识符"（舰队ID或其派生值）初始化 RNG，所有命中判定都从这个 RNG 取值。同舰队ID → 同种子 → 同命中序列 → 可复现；舰队ID变 → 种子变 → 序列变 → 不可复现。**

"固定种子现象"不是特殊的确定算法，**就是逐发 roll + 固定种子**。命中判定从头到尾只有一套机制，没有 N 分界，没有"期望伤害模式"。

---

## 一、决定性证据：三组控制变量对照

研究的突破口是识别出一个**之前从未被控制的变量：舰队ID**。围绕它做了充分必要双向证明：

| 实验 | 舰队ID | 结果 | 判定 |
|------|--------|------|------|
| 10斗牛停在基地边，开阳一个个来测（多局） | **不变**（未移动/未返航/未被毁） | 每艘船逐发伤害**跨局完全相同**，同局内各斗牛不同 | 同ID → 可复现 ✅ |
| 2混沌 vs CV3000，昨天 vs 今天（重新派遣） | **变了** | 伤害分布、时长**都变** | 异ID → 不复现 ✅ |
| 2混沌不回基地，连续和CV3000打第二场 | **不变**（同舰队实例） | 和第一场**完全相同**（时长+伤害分布） | 同ID → 可复现 ✅（本次） |

**充分性**（同ID→复现）+ **必要性**（异ID→不复现）双向都满足。

### 1.1 关键实验细节

- **10斗牛场景**：舰队停在对方基地边，结构值几乎无损（开阳打不动），服务器侧舰队实例持续存活 → 舰队ID恒定。开阳轮流入场测试，每局每艘斗牛的逐发命中伤害完全复现。
- **2混沌场景**：跨天测试时重新派遣舰队 → 舰队ID变 → 不复现；同一场战斗后不回基地立即再打 → 舰队ID不变 → 完全复现。

### 1.2 ★ 量化铁证：三方逐5秒命中序列对比（2026-07-02）

对"2艘混沌(有+25%命中) vs 1艘CV3000"场景，dump 三份战报的 HP曲线，提取逐5秒主炮命中发数序列（221个区间）：

| 对比 | 舰队ID | 相同率 | 判定 |
|------|--------|--------|------|
| 今天场1 vs 今天场2（同ID连续） | **同** | **100.00%**（221/221逐点） | 同ID→可复现 ✅ |
| 今天场1 vs 昨天（异ID重派） | **异** | 31.34%（随机水平） | 异ID→不复现 ✅ |
| 今天场2 vs 昨天（异ID重派） | **异** | 31.34%（随机水平） | 异ID→不复现 ✅ |

**前40区间并排**（直观可见）：
```
今天1: 0 0 0 3 2 1 3 0 3 0 4 2 1 2 0 4 0 4 2 2 3 0 4 2 2 4 0 2 0 2 1 1 4 0 3 1 2 4 0 3   ← 同ID
今天2: 0 0 0 3 2 1 3 0 3 0 4 2 1 2 0 4 0 4 2 2 3 0 4 2 2 4 0 2 0 2 1 1 4 0 3 1 2 4 0 3   ← 同ID (逐点一致)
昨天:  0 0 0 4 2 2 3 0 3 2 2 4 0 4 0 3 2 1 3 0 3 1 2 1 0 3 0 3 1 1 4 0 3 0 4 3 0 3 0 2   ← 异ID (31%=随机)
```

异ID的31.3%不是"部分复现"，是**纯统计巧合**——命中率相近时5秒区间命中数偶尔撞上的概率，属随机水平。

**复现精度**：逐5秒、逐发命中数级别。这是整个研究中最强的确定性证据。

## 二、为什么这个结论能化解之前所有矛盾

### 2.1 "N分界"是伪问题

04 文档反复纠结的核心未解问题："为什么 N≥2 可复现、N=1 随机？" 各种假设（连续期望、固定种子、无目标选择）都被反复推翻。

**H_id 的回答：这个 N 分界根本不存在。** 之前观察到的"N=1随机 vs N≥2确定"，是"舰队ID变 vs 不变"的混淆：

- 之前测 1V1 时，大概率每次重新派遣（ID变）→ 看起来"随机"
- 之前测 N≥2 单靶时（尤其10斗牛停在原地的实验），ID恰好稳定 → 看起来"确定"

两套机制本质相同：都是逐发 roll + 种子=舰队ID。差异只在种子是否稳定。

### 2.2 04-§3.5 的"Y2 排除"为何失效

04 文档排除"种子源自标识符"（Y2候选）的推理是：
> "若种子来自配置 → N=1和N≥2同是PvP，都该用配置种子，都该可复现。但实测N=1不可复现、N≥2可复现。"

这个推理有一个**隐藏的未验证前提**："N=1测试时舰队ID固定"。如果 N=1 测试时每次重新派遣（ID变），那 N=1 不可复现就**不能**用来排除种子假设。

更关键的是——用户确认"PVE打完全一样的目标场景经验判断不能很准确"，即 04 依赖的"NVN不可复现"经验判断本身就不可靠。**Y2 的排除不成立。** H_id 是 Y2 框架内从未被真正证伪的候选。

### 2.3 与已锁定铁证的一致性

| 已锁定铁证（03/04文档） | H_id 是否兼容 |
|---|---|
| 命中率公式=加法 `base×(1+k−dodge)` | ✅ roll 用的就是这个概率 |
| 伤害离散（每命中扣perHit整数倍） | ✅ roll 命中→扣整数倍，未命中→扣0 |
| NV1 可复现（10斗牛跨局相同） | ✅ 舰队ID恒定→种子恒定 |
| 命中发数=HP/perHit（物理必然） | ✅ 与种子无关 |
| 逐5秒命中序列有强自相关结构 | ✅ 来自CD(8.4s)与采样(5s)的拍频干涉，纯运动学效应 |

## 三、对引擎实现的战略含义（核心）

### 3.1 引擎命中机制：当前实现正确，无需改动

`simulator.ts` 的 `resolveAttack`（逐发 roll + 可注入 RNG）**结构与真实游戏一致**。

- `SeededRNG`（可复现随机源）的存在意义正是复现——引擎固定种子 = 服务器固定舰队种子，完全对应。
- **不存在**"N≥2切换到另一种确定算法"。04 文档纠结的"命中判定真实形式"答案是：就是逐发 roll。

### 3.2 04 文档待办清单大幅瘦身

以下项**全部撤销**（H_id 让它们失去意义）：

| 撤销项 | 撤销理由 |
|---|---|
| ❌ "命中判定的具体实现形式" | 就是逐发roll，没有别的 |
| ❌ "N≥2切换确定模式" | 伪问题，N分界不存在 |
| ❌ "连续期望伤害分支" | 已废（伤害离散）+ 无需（就是roll） |
| ❌ "base=区间下限 vs 中值"之争 | 那是种子固定后的拟合伪值之争；base就是正常roll的中值 |
| ❌ "多舰批次不同步"假说（03-§3） | 无工程必要、无证据，早已放弃 |

### 3.3 验证范式转换：从"比特复现"到"统计等价" ⭐

这是最大的实操转变。

**旧目标（不可能）**：逐发比特级复现某份战报——要求猜中服务器的种子（种子=舰队ID，拿不到）。

**新目标（可达成）**：

> 同配置用**不同种子**跑 N 场，检查统计量的**分布**是否吻合实测值。

校准锚点（战报提供的）：
- `+25%命中 → DPS比1.2456`（命中率公式验证）
- 逐5秒命中发数的**分布**（均值/方差/自相关结构）
- 总命中数 = HP/perHit（物理必然，必须精确）

### 3.4 可直接推进的下一步（不再被种子阻塞）

1. **接入客户端配置表**（787舰/583武器）让引擎用真实数据
2. **统计验证管线**：批量模拟 + 分布对比
3. **激活下一层机制**：拦截/反导、目标优先级、子系统摧毁

## 四、残留的次要问题（不影响推进）

1. **种子派生方式**：种子=舰队ID本身，还是 hash(舰队ID+ship_uid)，或别的派生？——对模拟器无影响（我们用任意种子做统计），但对"完全复现某场战报"有影响（且后者本就不是目标）。
2. **同局内各舰伤害不同的机制**：每艘船按开火相位消耗RNG的不同位置（相位错开），或种子含ship_uid。需逐舰数据才能区分，但这是细节问题。
3. **NVN场景的目标选择层随机**：若 NVN 同ID连续打两场，是否仍可复现？若不可复现，说明目标选择层有独立随机源。这是次要问题，不影响单靶场景的引擎实现。

## 五、方法论沉淀

本轮的关键突破不是新数据，而是**识别出之前所有研究都遗漏的控制变量（舰队ID）**。教训：

1. **"已排除的假设"要检查其隐藏前提**——Y2 的排除依赖"N=1测试时ID固定"这个从未验证的前提。
2. **经验判断不可作为铁证**——04 依赖的"NVN不可复现"是模糊经验，用户明确表示不可靠。
3. **控制变量是王道**——10斗牛实验的可复现性，本质是它无意中控制了ID变量。识别出这个变量后，主动设计对照（同ID连续 vs 异ID重派）一举定案。
