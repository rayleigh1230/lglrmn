## 三、DPH/命中率/属性的真实来源 —— 蓝图科技串（★已完全打通）

cfg_weapon 只有时序和类型，**不含伤害数值**。实测确认：DPH、命中率、抵抗、护盾、暴击等数值来自**蓝图装配的模块科技等级**，经 `cfg_system_enhance` → `cfg_system_effect` 链路查得。

### 3.1 数据链路（★2026-07-02 验证打通）

```
战报块0 (team_battle_data) 的科技串:
  "8020109,2,1132,5,1,1131,5,3,1210,5,4,1211,5,8,8890,1;..."
       ↓ 解析（三元组步长3）
  slotId=8020109, 装载: (subType=2,techId=1132,lv=5), (subType=1,techId=1131,lv=5), 
                       (subType=8,techId=8890,lv=1), ...
       ↓ techId 是 cfg_system_enhance 的 SYSTEM_EFFECT_PREFIX
  cfg_system_enhance[802010908].SYSTEM_EFFECT_PREFIX = 8890
       ↓ SYSTEM_EFFECT_PREFIX + level → cfg_system_effect 的 key
  cfg_system_effect[889001] (8890 + "01"=level1) = 抗冲击结构, EFFECT_ID=10, PARAM=500
       ↓ EFFECT_ID=10 → 结构强化机制; PARAM=500 → 数值
  → "舰船结构值提升{101}%"
```

### 3.2 科技串格式（★已精确解码）

每个 `;` 分隔一个槽位，槽位内是**三元组** `(subType, techId, level)`：
```
slotId, subType1, techId1, level1, subType2, techId2, level2, ...
```
- CV3000 槽 `8020109`：装 5 个三元组 `(2,1132,5)(1,1131,5)(3,1210,5)(4,1211,5)(8,8890,1)`
- subType = 子系统类型（1-20），techId = system_enhance 的 SYSTEM_EFFECT_PREFIX，level = 强化等级

### 3.3 techId → system_effect 的等级映射

`SYSTEM_EFFECT_PREFIX(P) + level(L)` 拼成 system_effect 的 key：
- level 1 → key = `P + "01"` (如 8890→889001)
- level 5 → key = `P + "05"` (如 1132→113205)
- 部分高等级有独立变体（820601/820602）

### 3.4 ★铁证验证：CV3000 +33% HP

```
CV3000 装配的结构强化（EFFECT_ID=10）:
  techId=1132 (焊接工艺强化II) lv5  → 焊接工艺强化
  techId=1131 (龙骨结构增强II) lv5  → 龙骨结构增强  
  techId=8890 (抗冲击结构) lv1      → EFFECT_PARAM=500
基础结构 278340 × 1.33 = 370192 = 战报实测值 ✅
```
（1132/1131 的 level5 数值需查 tech_bp_skill_level 等级表，PARAM=0 是因为基础条目不含等级表值；8890 PARAM=500 直接生效）

### 3.5 关键认知修正

- ❌ 旧认知：techId 直接是 system_effect 的 key
- ✅ 新认知：techId = `cfg_system_enhance` 的 `SYSTEM_EFFECT_PREFIX`，需先查 enhance 表定位，再拼 level 查 system_effect

### 5.3 引擎对接

```typescript
// 从 cfg_ship_slot 提取某舰的所有武器
function slotsToWeapons(shipId): WeaponSlotDef[] {
  return Object.entries(cfg_ship_slot)
    .filter(([k]) => k.startsWith(shipId))
    .filter(([,v]) => v[0] === 1 || v[0] === 2)   // 只取武器槽
    .map(([k,v]) => ({
      slotId: k,
      weaponId: v[2],
      category: v[0],        // 1=主 2=副
      isWeapon: true,
    }));
}
```

### 7.1 战报三块结构

```
{team_id: [块0, 块1, 块2]}
```

**块0 — 舰队配置**（含科技串，★★★属性来源）：
```
[userid, ship_id, ship_type, 0, bp_id?, 0, [coords], "模块清单", "★科技串", ...]
```

**块1 — HP 曲线**（逐5秒采样）：
```
"type,time,struct_hp,shield_hp#..."
  type=5:  常规HP采样
  type=97: 击毁事件 (struct_hp=ship_uid)
  type=99: 战斗结束
```

**块2 — 舰船+攻击事件**：
```
[玩家信息, [舰船记录], [攻击事件]]
舰船记录: [ship_id, ship_uid, max_hp, userid, ..., cur_hp, ...]
攻击事件: [attacker_uid, weapon_id, [], [
  [event_type, [[target_uid, val1, val2], ...]]
]]
  event_type=0: 命中/伤害 (val1=dph累计, val2=伤害值)
  event_type=2: 持续输出统计 (val1=begin_time, val2=累计伤害)
  event_type=5: 击毁时刻 (val1=begin_time, val2=end_time)
  event_type=8: 伤害统计汇总
```
