# 11-数据流与表间关联（数据层设计依据）

> 日期：2026-07-02
> 目的：理清从客户端配置表到引擎 Ship 的完整数据流，作为数据层实现的权威依据
> 基础：27 张配置表已全部 dump（见 docs/10）

---

## 〇、核心结论

**配置表提供"骨架 + 加成机制"，不提供武器基础 dph。** 基础 dph 来自战报攻击事件（服务器计算结果）。数据层的职责是：

1. ✅ 从配置表解析**船型骨架**（HP/速度/槽位/武器循环）
2. ✅ 从科技串解析**加成机制**（结构强化/命中/暴击/闪避/抵抗/护盾）
3. ⚠️ 基础 dph 需从战报反推，或后续 dump 武器伤害表

## 一、表间关联图（外键关系）

```
cfg_ship (船型骨架)
  └─ ship_id ──→ cfg_ship_slot (槽位装载)
                    └─ weapon_id ──→ cfg_weapon (武器循环参数)
                                      └─ WEAPON_TYPE ──→ 伤害模型映射
  └─ ship_id ──→ cfg_ship_system (子系统结构)
                    └─ system_id ──→ cfg_system_enhance (可强化选项)
                                      └─ SYSTEM_EFFECT_PREFIX ──→ cfg_system_effect (效果+数值)
                                                                    └─ EFFECT_ID ──→ cfg_effect_def (机制字典)
```

### 1.1 主键/外键清单

| 表 | 主键 | 格式 | 外键指向 |
|---|---|---|---|
| cfg_ship | ship_id | 5位(80201) | — |
| cfg_ship_slot | slot_id | 9位(802010101) = shipId+row+col | → cfg_weapon[weapon_id] |
| cfg_weapon | weapon_id | 5位(18021) | — |
| cfg_ship_system | system_id | 7位(8020104) = shipId+slot | — |
| **cfg_system_enhance** | enhance_id | **9位(802010901) = shipId+slot+optIdx** | → cfg_system_effect[PREFIX+level] |
| cfg_system_effect | effect_id | 6-7位(889001) = PREFIX+level | → cfg_effect_def[EFFECT_ID] |
| cfg_effect_def | EFFECT_ID | 数字(10/12010) | — (机制字典) |

## 二、科技串 → 数值 的完整解析链路（★已验证）

### 2.1 科技串格式（三元组）

```
"8020109,2,1132,5,1,1131,5,3,1210,5,4,1211,5,8,8890,1"
   ↑     ↑  ↑   ↑  
   slotId subType techId level  (循环,步长3)
```

- `slotId`(7位) = shipId(5) + slot(2)，对应 cfg_ship_system
- `techId` = cfg_system_enhance 的 SYSTEM_EFFECT_PREFIX
- `level` = 强化等级（1-5）

### 2.2 解析步骤

```
Step1: 三元组拆分
  (subType=2, techId=1132, level=5)

Step2: techId → system_effect key
  key = techId + level补零 = "1132" + "05" = "113205"
  (注: level1用"01", level5用"05")

Step3: 查 cfg_system_effect[key]
  113205 → {EFFECT_ID:10, NAME:"焊接工艺强化II", EFFECT_PARAM_LEVEL:"1,200;2,500;3,800;4,1100;5,1400"}
  level5 取 PARAM_LEVEL 的 "5,1400" → 数值=1400

Step4: EFFECT_ID → 机制(查 cfg_effect_def)
  EFFECT_ID=10 → "舰船结构值提高" → 结构强化(万分比)

Step5: 数值应用
  1400/10000 = 14% 结构提升
```

### 2.3 ★铁证验证（CV3000）

```
装配的结构强化:
  techId=1132(焊接II) lv5 → PARAM_LEVEL[5]=1400 → 14%
  techId=1131(龙骨II) lv5 → PARAM_LEVEL[5]=1400 → 14%  
  techId=8890(抗冲击) lv1 → EFFECT_PARAM=500 → 5%
  合计 = 33%
基础 278340 × 1.33 = 370192 = 战报实测 ✅ (误差0)
```

**关键规则**：
- 多个同类 EFFECT **加法叠加**（1132+1131+8890 = 3300）
- PARAM 单位是**万分比**（/10000），3300 = 33%
- 有 PARAM_LEVEL 的按等级查表；无 PARAM_LEVEL 的用 EFFECT_PARAM

## 四、武器 dph 的来源问题（★重要缺口）

### 4.1 现状

配置表**不含武器基础 dph**：
- cfg_weapon：只有循环参数（CD/前摇/类型），无 dph
- cfg_ship_system：只有子系统 HP，无 dph
- cfg_system_effect：只有"伤害提升X%"加成，无基础值

### 4.2 dph 的真实来源

基础 dph 在**战报攻击事件**里（服务器算好的结果）：
```
战报块2 攻击事件: [target_uid, weapon_id, [], [[0, [[attacker, 累计伤害, 单次伤害]]]]]
                                                                  ↑ 这里有
```

### 4.3 对数据层的影响

- **战报导入模式**（importFromBattleReport）：dph 直接从攻击事件反推（单次伤害 = dph − 抵抗），✅ 可用
- **自由配置模式**（UI装配）：缺少基础 dph 表，⚠️ 需额外 dump 或建立经验表

### 4.4 后续补 dump 方向

可能的 dph 来源表（待探测）：
- `Tb_cfg_weapon` 可能还有未 dump 的字段（如 DAMAGE_BASE）
- 或在 `Tb_cfg_ship_system` 的扩展字段
- 或武器 dph 本身就编码在 weapon_id 里（需反推）

## 五、数据层必须处理的核心数据结构

### 5.1 船型骨架（来自 cfg_ship + cfg_ship_slot + cfg_weapon）

```typescript
interface ShipHull {
  shipId: string;
  name: string;
  targetClass: TargetClass;      // 由 ship_class 码 + name 推断
  baseStructure: number;         // cfg_ship[4]
  speed: number;                 // cfg_ship[5]
  sizeClass: number;             // cfg_ship[21]
  slots: WeaponSlot[];           // 来自 cfg_ship_slot
}

interface WeaponSlot {
  slotId: string;
  category: number;              // cfg_ship_slot[0]: 0=模块 1=主武 2=副武
  weaponId: number;              // cfg_ship_slot[2]
  weapon?: WeaponDef;            // 来自 cfg_weapon
}
```

### 5.2 武器循环（来自 cfg_weapon）

```typescript
interface WeaponDef {
  weaponId: number;
  damageType: DamageType;        // WEAPON_TYPE 映射
  delivery: WeaponDelivery;      // WEAPON_TYPE 映射
  category: WeaponCategory;      // WEAPON_TYPE 映射
  cooldown: number;              // CD_TIME/1000
  fireDuration: number;          // (ATTACK_INTERVAL-CD_TIME)/1000
  lockOnTime: number;            // INIT_PERPARE_TIME/1000
  weaponType: number;            // 原始 WEAPON_TYPE
}
```

### 5.3 蓝图强化（来自科技串 → system_enhance → system_effect）

```typescript
interface BlueprintEnhancement {
  slotId: string;
  techId: number;                // SYSTEM_EFFECT_PREFIX
  level: number;
  effects: ResolvedEffect[];     // 解析后的效果列表
}

interface ResolvedEffect {
  effectId: number;              // EFFECT_ID
  mechanism: string;             // 来自 effect_def 的中文名
  value: number;                 // PARAM 或 PARAM_LEVEL[level]
  unit: 'permille' | 'absolute'; // 万分比 or 绝对值
}
```

### 5.4 最终 Ship（骨架 + 强化汇总）

```typescript
// resolveBlueprint(hull, enhancements): Ship
interface ResolvedShip {
  // 骨架
  shipId, name, targetClass, cell;
  // 强化后数值
  structure: number;             // baseStructure × (1 + Σ结构强化)
  resistance: number;            // 来自抵抗类 EFFECT
  shield: number;                // 来自护盾类 EFFECT
  dodge: number;                 // 来自闪避类 EFFECT
  // 武器（含强化）
  weapons: WeaponSystem[];       // 每个武器含 dph加成/hitBonus/critRate
}
```
