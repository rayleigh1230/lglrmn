# 蓝图火力计算研究

> 日期：2026-07-03
> 状态：✅ **公式已完全突破（frida dump 表达式树 AST 还原），对舰/攻城精确匹配，4武器10锚点全过**

## ★ 突破总结（2026-07-03 新增）

通过 frida 注入 `data.ship_attr_calc.BlueprintAttrCalc`，用 `prepare_by_ship_id` 创建计算器，dump 出 `get_weapon_ship_dps_calc` 返回的表达式树 AST（LazyCalculatable），精确还原公式：

### 精确公式（4武器全匹配，零误差）

```
周期秒 period = fireDuration(action[4]/1000) + cooldown(CD_TIME/1000)
通用因子 cycleFactor = attackRounds(action[0]) × attackCount(action[3]) × 60 × installNum(slot[1]) / period

【对舰火力】= max(dph - 10, dph × 0.1) × cycleFactor
  （DEFAULT_ARMOR=10，面板默认抵抗值）

【攻城火力】= (dph × DESTROY_COEF / 100) × cycleFactor
  （仅 DESTROY_COEF > 0 的武器贡献）

【防空火力】= 防空单发 × cycleFactor / 100 × antiaircraftRatio(slot[4]) × fixedHitRate(0.15)
  防空单发 = base(action[2]) + 对空基础单发增加(module_effect EID=12300)
  （仅 antiaircraftRatio > 0 的武器贡献）
```

### ★ 防空单发构成（2026-07-03 补充确认）

防空单发与对舰单发**机制不同**：
- 对舰单发 = `max(base - 10, base × 0.1)`（穿抗，减抵抗值10）
- 防空单发 = `base + 对空基础单发增加`（**不穿抗**，加固定增伤词条）

`对空基础单发增加`来自 `cfg_module_effect[weaponId+"01"]`，EFFECT_ID=12300，PARAM=固定增伤值。
- FG300主炮13011：base=30 + 增伤50 = 80 ✓（表达式树常量80，锚点1851精确匹配）
- ST59武器15017：base=40 + 增伤0 = 40 ✓（无对空增伤词条）

### ★★ 防空专用 period 缩放（2026-07-03 澄海级突破）

**防空 period 与对舰 period 不同！** 受防空专用词条缩放：
- 防空 duration' = duration × (1 - EID=12311/100)  ← **防空高效打击**（对空攻击持续时间下降）
- 防空 cd' = cd × (1 - EID=12306/100)              ← **防空高速高效循环**（对空冷却时间下降）
- 防空 period = 防空 duration' + 防空 cd'

`EID=12306`/`EID=12311` 来自 `cfg_module_effect[weaponId+"02"/"03"]`，PARAM=下降百分比。

实测验证（3武器零误差）：

| 武器 | EID=12300 | EID=12306 | EID=12311 | 防空单发 | 防空period | 防空火力 |
|---|---|---|---|---|---|---|
| FG300-13011 | 50 | 0 | 0 | 80 | 7.0(无缩放) | 1851 ✓ |
| 澄海-13124 | 60 | 40 | 40 | 80 | 6.0(2.4+3.6) | 1440 ✓ |
| 澄海-13122 | 10 | 10 | 10 | 25 | 5.4(2.7+2.7) | 375 ✓ |

**防空三件套**（module_effect 的舰船特有词条，非强化项）：
- EID=12300 对空基础单发增加 → 防空单发 += PARAM
- EID=12306 对空冷却时间下降 → 防空cd ×= (1 - PARAM/100)（防空高速高效循环）
- EID=12311 对空攻击持续时间下降 → 防空dur ×= (1 - PARAM/100)（防空高效打击）

### 高速循环/冷却策略不计入面板（对舰/攻城）

面板对舰/攻城火力用**基础CD**（CD_TIME/1000）。大量冷却类效果（EID=12041系统散热优化、EID=12141快速输出等）
都是**战斗中动态触发的策略**（带 {T}秒持续 / {C}秒冷却），不纳入静态面板火力计算。
FG300空强化下 cd_time=4.0=CD_TIME/1000，无任何冷却强化影响。

注：防空的 EID=12306/12311 是**舰船特有静态词条**（写在 module_effect 里，非战斗策略），所以计入面板。
与战斗策略（EID=12041等）本质不同。

### 表达式树 AST mode 语义
- mode=1: 常量叶子
- mode=2: 乘法
- mode=3: 除法
- mode=5: 加法
- mode=7: max(a, b)

### 关键数据字段
- `cfg_weapon_action[key]` = `[action[0], action[1], dph, action[3], duration_ms]`
  - action[0] = attackRounds（每轮攻击次数，如 FG300=3）
  - action[2] = dph（单发基础伤害）
  - action[3] = attackCount（攻击次数倍数，通常1或2）
  - action[4] = fireDuration（持续开火毫秒）
- `cfg_ship_slot[slotId]` = `[cat, installNum, weaponId, ..., antiaircraftRatio, ...]`
  - slot[1] = installNum（安装数量）
  - slot[4] = antiaircraftRatio（防空比例，如200）
- `cfg_weapon` = `{ DESTROY_COEF, AIRCRAFT_COEF, CD_TIME, WEAPON_TYPE, ... }`

## 以下为早期研究记录（已突破，保留作参考）



## 一、游戏内部火力计算链路（已确认）

### 计算器类
```
data.ship_attr_calc.BlueprintAttrCalc
```

### 计算入口
```python
calc = BlueprintAttrCalc.create_prepared(ship_id, enhancements, modules)
# enhancements: 强化效果(frozenset格式，非字符串)
# modules: 已装配系统串(如"4100302,4100303,...") 或 None

# 三类火力
calc.get_ship_dps('ship_firepower')      # 对舰火力
calc.get_ship_dps('ship_aa_firepower')   # 防空火力
calc.get_ship_dps('ship_siege')          # 攻城火力
```

### 公开接口（ship_utils 模块）
```python
ship_utils.get_ship_dps_by_detail(ship_id, enhancements, modules, dps_type)
ship_utils.get_ship_dps(ship_id, dps_type, ship_record)  # 需要玩家数据
ship_utils.get_ship_dps_for_types(ship_id, dps_types, ship_record)
```

### 表名常量
```python
MA_SHIP_DPS = 'ship_firepower'           # 对舰
MA_SHIP_AIR_DPS = 'ship_aa_firepower'    # 防空
MA_SHIP_DESTROY_COEF_DPS = 'ship_siege'  # 攻城
```
这些是动态计算标识，**不是独立配置表**（get_table 返回 None）。

### BlueprintAttrCalc 关键方法
- `get_ship_dps(dps_type)` — 总火力
- `get_ship_dps_calc(dps_type)` — 火力计算明细
- `get_weapon_air_defend_dps(...)` — 单武器防空火力
- `get_weapon_destroy_coef_dps(...)` — 单武器攻城火力
- `get_aircraft_dps(...)` — 舰载机火力
- `get_air_defend_fixed_hit_rate(...)` — 防空固定命中率
- `get_module_total_dps(...)` — 模块总火力

## 二、未突破的难点

1. **create_prepared 参数**：
   - `enhancements` 需要 frozenset，不是字符串
   - effects_enhanced 字符串（"slotId,optIdx,techId,level;..."）需要转换成 frozenset 格式
   - 空强化时 `frozenset()` 能创建成功，但 `get_ship_dps` 内部某处返回 None

2. **Hook 失败**：
   - monkeypatch `get_ship_dps` 后，游戏蓝图面板 60 秒内未触发调用
   - 可能游戏用缓存，或用 `get_ship_dps_calc`/其他入口

3. **攻城火力**：无通用公式，由游戏内部 DESTROY_COEF 系数计算

## 四、当前 UI 使用的近似公式（blueprintCalc.ts）

```
每分钟发数 = shotsPerCycle / (fireDuration + cooldown) × 60
单发处理后 = max(dph - 10, dph × 0.1)  // 实弹，抵抗10
对舰火力 = Σ 对舰武器: 单发处理后 × 每分钟发数 × 100%命中
防空火力 = Σ 所有武器: 单发处理后 × 每分钟发数 × 命中口径(舰载15%/机载60%)
攻城火力 = Σ 攻城武器(WT=8): 单发处理后 × 每分钟发数 × 100%
```
注意：此公式与游戏实际值有偏差，待后续突破后校准。
