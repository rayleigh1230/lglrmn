## 二、缺陷清单与修复（5 项）

### 缺陷1【高·已修复】调校(adjust)效果出口错误

**客户端真相**：调校槽（EFFECT_TYPE=2 且有 ADJUST_PROB）有自己的 EFFECT_ID，但属性产出走**父强化** EFFECT_ID 通道。`is_enhance_influence_effect_value` 对调校做重定向：
```python
if enhance_id in SYSTEM_ADJUST_IN_ENHANCE:
    enhance_id = SYSTEM_ADJUST_IN_ENHANCE[enhance_id]   # adjust → 父强化
effect_id = get_enhance_effect_id(enhance_id)
```
然后 `calc_effect_add` 用**父 EID** 查 `cfg_weapon_num_attr` 决定通道。

**数据实证**（2254 调校槽，`cfg_system_enhance` 全表统计）：
- 79%（1780 槽）调校自身 EID == 父 EID（如结构 EID=10）—— 行为不变
- 8%（181 槽）两者不同：调校自身是技能触发器（EID 13005 等，不在 weapon_num_attr），父强化才是真正影响属性的（如 EID 12141 duration ratio_del）
- 59%（1326 槽）两者都是纯技能触发器 —— 正确归零

**典型例**：ST59 调校 `603010140`（自身 EID=13005「快速输出」技能触发器，PARAM=10000）→ 父 `603010110`（EID=12141「缩短冷却与打击时间」，PARAM=80，duration ratio_del）。旧实现把 13005 推进 effectList 被自动过滤为 0 → 调校零产出。

**修复**（`packages/engine/src/data/tuneSystem.ts`）：
- `TuneSlot` 新增 `parentEffect`（父强化 effect，属性产出通道）；`effect` 保留供 UI 显示
- `computeTuneBonus` 用**父 EID** 产 EffectEntry，数值按 calc_effect_add B类公式 `parent.PARAM × cur_level / max_level`（max_level=len(ADJUST_PROB)=10），A类查父 PARAM_LEVEL
- 测试 `tune-enhance-link.test.ts` 更新为正确语义

**commit**：`90b3b7e`

### 缺陷2【高·已修复】巅峰 COMMON_EFFECT 变体舰归一化缺失

**客户端真相**（`get_peak_effect`）：
```python
# COMMON (field[0]) 归一化到基础舰
ship_id = (ship_id // 10) * 10 + 1
common_dict = parse(SHIP_PEAK_LEVEL[ship_id][peak_level-1].COMMON_EFFECT)
# EXCLUSIVE (field[1]) 用字面 ship_id
exclusive_dict = parse(SHIP_PEAK_LEVEL[orig_ship_id][peak_level-1].EXCLUSIVE_EFFECT)
return (common_dict, exclusive_dict)
```

**数据实证**：`cfg_ship_peak_level` 共 174 艘船，其中 **85 艘是变体**（如 FG300 的 30102/30103）。变体行 field[0] **恒为空**（1700 行实测），COMMON 数据只存在基础舰行。旧实现用字面 shipId 拼 key → 变体舰查不到 COMMON → 巅峰结构/移速加成全部丢失。

**修复**（`packages/engine/src/data/peakLevel.ts`）：
- 新增 `toMainShipId(shipId)`：`(n//10)*10+1`
- `getPeakSnapshot`：field[0] 用 mainShipId 查表，field[1] 用字面 shipId
- 测试 `peak-variant-normalization.test.ts`：41301基础/41302变体对照，归一化后变体继承基础 COMMON 结构加成(1730)

**commit**：`3112fcd`

### 缺陷3【高·已修复】结构分层不符合 get_ship_hp

**客户端真相**（`common.ship_utils.get_ship_hp`）：
```python
ship_hp = get_after_system_effect_value(enhancements, ..., ship_hp_base)  # 强化 EnhanceBasic 作用于 base
ship_hp += get_blueprint_ex_add_values(record)['ship_hp_add']             # 技术值绝对值
final_hp = int(ship_hp + ship_hp_base × Σmodule_EID10_ratio / 100)       # 模块比例作用于裸base，末尾加，不放大
```

**关键点**：模块 EID=10 比例作用对象是**裸 base**，且**不**被系统强化比例放大。

**旧实现（错误）**：
```ts
skeleton = base + moduleBonus
final = floor(skeleton × (1 + 强化比例))   // 把 moduleBonus 也放大了
```

**修复**（`packages/engine/src/data/blueprintResolver.ts`）：
```ts
enhancedBase = floor(base × (1 + 强化比例))         // 强化仅作用于 base
finalStructure = enhancedBase + 巅峰绝对值 + 版本号绝对值 + moduleStructureBonus  // 模块末尾加，不放大
```

**验证**：module+enhance 共存场景（80201 base=278340, 强化14%, module=32484）：
- 旧放大公式 = 354339（偏大 4548 = 模块部分 32484×14%）
- 新对齐公式 = 349791（正确）
- CV3000 锚点（370192，module=0）不受影响；module-structure 锚点（无强化）不受影响

**commit**：`185648b`

### 缺陷4【中·已修复】巅峰奖励 EXCLUSIVE 语义订正

**客户端真相**：field[1] 是**巅峰强化奖励（EXCLUSIVE_EFFECT）**，enhanceId 是 EFFECT_TYPE_ENHANCE_ADD（optIdx 70/71），不是"调校"。真正的调校（ADJUST，optIdx 31-43）在 tuneSystem.ts。

**数据实证**：optIdx 70/71 的 EFFECT_TYPE 是 3/5（ENHANCE_EX/ADD），共 606 条。等级直接取 EXCLUSIVE dict 的值。

**修复**：`peakLevel.ts` 顶部注释订正；field[1] 查表用字面 shipId（已在缺陷2修复中落实）。`docs/13` §2.5/§2.8 同步订正。

**commit**：`3112fcd`（注释）+ `185648b`

### 缺陷5【中·已审计】单位一致性

**客户端真相**（docs/16 §2.3 + EnhanceBasic.expression）：
- `add_ratio`（effectList 通道）单位是**百分数**（40.0 = 40%），公式 `(100+add_ratio)/100`
- 面板比例字段（结构/命中/闪避/暴击等）按游戏 UI 惯例用**万分比**（1500 = 15.0%）

**审计结论**：模拟器两层单位**各自正确**，非计算 bug：
- `effectList.ts:getEnhanceAdd` 直接累加 e.value 进 addRatio（百分数 ✓，对齐 calc_effect_add）
- `blueprintResolver.ts` 面板字段 `×100` 转万分比（显示口径 ✓，客户端面板同口径）
- `phases/hit.ts` 战斗公式 `base × (1 + bonus)` 用分数（0.15=15%）—— 由调用方转换

**修复**：在 `ResolvedBlueprint` 接口顶部补**单位约定文档**，明确：面板字段=‱（显示）、effectList=%（计算）、结构类=绝对值。避免后续混淆。

**commit**：`185648b`

## 三、EFFECT_TYPE / UNLOCK_TYPE 建模（补全）

`rawTypes.ts` 补枚举语义注释（数据实证自 28198 行 `cfg_system_enhance`）：

| 字段 | 值 | 客户端枚举 | 含义 |
|---|---|---|---|
| EFFECT_TYPE | 缺失 | EFFECT_TYPE_ENHANCE | 普通强化项（optIdx 01-11） |
| | 2 | ENHANCE/ADJUST | 调校槽（optIdx 31-43，带 ADJUST_PROB） |
| | 3 | EFFECT_TYPE_ENHANCE_EX | 巅峰奖励 EXCLUSIVE（optIdx 70/71） |
| | 5 | EFFECT_TYPE_ENHANCE_ADD | 巅峰奖励 EXCLUSIVE（optIdx 70/71） |
| UNLOCK_TYPE | 缺失/0 | UNLOCK_TYPE_NONE | 普通槽 |
| | 2 | UNLOCK_TYPE_RE_TECH | 逆向科技/武器技术解锁槽（optIdx 4-12 浮动） |

客户端 `get_enhancement_type` 据此三分类：SYSTEM_ENHANCEMENT / SYSTEM_EXTEND / SYSTEM_ADJUST。

## 四、不做的事（明确边界）

按"计算核心正确性"范围，以下**未实装**（客户端有但模拟器不需要/超出范围）：

| 客户端能力 | 不做原因 |
|---|---|
| `get_peak_auth_info`（巅峰授权道具+任务解锁） | 玩家进度层，不影响战斗数值 |
| 多槽位 `SystemScheme` 预设存档（SLOT_INDEX 多套调校） | 存档/UI 层 |
| `gameplay`/`collection` 双表（SHIP_BLUEPRINT_GAME_PLAY 等） | 多人场景路由，单机模拟不需要 |
| `cal_bp_record_use_research_point`（精确技术值会计） | 数据会计层；模拟器由调用方传入 techPoints |
| `get_aircraft_dps`（无人机/载机递归 DPS） | 已部分支持（group_num），递归增强过滤待后续 |
