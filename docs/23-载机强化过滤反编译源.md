### 修正缩进版（正确控制流）

```python
def _filter_drone_enhance(cur_enhance_dic):
    filtered_dic = {}
    for system_id, enhance_dic in six.iteritems(cur_enhance_dic):
        filtered_enhance = {}
        for enhance_id, level in six.iteritems(enhance_dic):
            if not blueprint_utils.is_enhance_influence_effect_value(enhance_id):
                filtered_enhance[enhance_id] = level
                continue
            effect_id = blueprint_utils.get_enhance_effect_id(enhance_id)
            system_effect_id_lst = SYSTEM_EFFECT_ENHANCE_DATA.get(effect_id, [])
            has_drone_effect = False
            for system_effect_id in system_effect_id_lst:
                effect_config = Tb_cfg_system_effect.get(system_effect_id)
                if effect_config and effect_config[Tb_cfg_system_effect.EFFECT_ID] in common_definition.DRONE_EFFECT_IDS:
                    has_drone_effect = True
                    break
            if not has_drone_effect:                          # ← 移出内层 for
                filtered_enhance[enhance_id] = level
        if filtered_enhance:                                   # ← 移出内层 for
            filtered_dic[system_id] = filtered_enhance
    return filtered_dic                                        # ← 移出最外层 for
```

### 逻辑说明

1. **遍历母舰强化字典** `cur_enhance_dic`（按 system_id 分组的 `{enhance_id: level}`）。
2. **不影响数值的强化直接保留**：`is_enhance_influence_effect_value(enhance_id)` 返回 False 的强化（如纯 UI/标记型）原样透传。
3. **会影响数值的强化做触达判定**：
   - `get_enhance_effect_id(enhance_id)` 取该强化的 effect_id（经调校→父强化重定向后的 prefix）。
   - 查 `SYSTEM_EFFECT_ENHANCE_DATA[effect_id]` 得该 effect 关联的 system_effect_id 列表。
   - 逐个查 `Tb_cfg_system_effect[se_id].EFFECT_ID`，若任一 ∈ `DRONE_EFFECT_IDS`（{2020,2021,2022,2023}）→ 标记 `has_drone_effect=True`。
4. **触达载机的强化被剥除**（不加入 filtered_enhance），防止载机递归时又触发"搭载载机"导致无限套娃。
5. **未触达的强化透传给载机**（如"载机伤害提升""武器伤害"等）。

### 关键常量/表

| 名称 | 含义 | TS 对应 |
|---|---|---|
| `cur_enhance_dic` | 母舰强化字典 `{system_id: {enhance_id: level}}` | 由 `blueprint.effects[].tech` 重建 |
| `SYSTEM_EFFECT_ENHANCE_DATA` | `{effect_id: [system_effect_id,...]}`，2362 条 | `store.systemEffectEnhanceData` |
| `DRONE_EFFECT_IDS` | `{2020,2021,2022,2023}` | `DRONE_EFFECT_IDS`（blueprintCalc.ts:334） |
| `Tb_cfg_system_effect` | 系统效果主表 | `store.systemEffect` |

```python
def get_aircraft_dps(self, dps_type, all_effects_list, cur_enhance_dic):
    total_dps = 0
    for effect_info in all_effects_list:
        _system_id = effect_info.system_id
        effect_config = effect_info.effect_record
        effect_id = effect_config[Tb_cfg_system_effect.EFFECT_ID]
        if effect_id in common_definition.DRONE_EFFECT_IDS:
            value = effect_config[Tb_cfg_module_effect.EFFECT_PARAM]
            ship_id = value / 100
            num = value % 100
            bp_attr_calc_obj = BlueprintAttrCalc()
            bp_attr_calc_obj.prepare_by_ship_id(ship_id)
            filtered_enhance_dic = cls._filter_drone_enhance(cur_enhance_dic)
            bp_attr_calc_obj.set_mother_ship_info(_system_id, filtered_enhance_dic)
            dps = bp_attr_calc_obj.get_ship_dps(dps_type)
            total_dps += dps * num
    return total_dps                                        # ← 移出 for
```

### 逻辑说明

遍历母舰的 `all_effects_list`，对每个"搭载载机"效果（EFFECT_ID ∈ DRONE_EFFECT_IDS）：
1. 解码载机 ship_id 和数量（`PARAM/100`、`PARAM%100`）。
2. 创建子载机计算对象，按 ship_id 准备（`prepare_by_ship_id`）。
3. **`_filter_drone_enhance(cur_enhance_dic)`** 过滤母舰强化 → 传给子载机。
4. 子载机算自身 DPS（`get_ship_dps`，含其自身武器 + 透传的母舰强化）。
5. 累加 `dps × num`。

### TS 映射

TS 的 `computeAircraftDps`（blueprintCalc.ts:365）已实现步骤 1/4/5，
**缺失步骤 2-3 的强化透传**（当前传空 effectList）。修复点即在载机 `computeFirepower` 调用处。

# File: get_effect_list.pyc (Python 3.11)

all_system_effect_list = []

def extend_effect_list(effect_id_list = None, _system_id = None, _cur_level = None, _max_level = None):
    for system_effect_id in effect_id_list:
        main_system_effect_id = (system_effect_id // 100) * 100 + 1

        system_effect_id_lst = SYSTEM_EFFECT_ENHANCE_DATA[effect_id]
        max_level = blueprint_utils.get_enhancement_max_level(enhance_id)
        cur_level = min(level, max_level)
        extend_effect_list(system_effect_id_lst, system_id, cur_level, max_level)
        for module_info in self.cur_modules:
            module_id = module_info[0]
            slot_id = module_info[1]
            system_id = slot_id / CfgShipField.Prefix.PREFIX_SYSTEM_ID_TO_SLOT

### 逻辑说明

把 `cur_enhance_dic`（强化字典）重建为 `all_system_effect_list`（效果列表）：
1. 遍历每个会影响数值的强化项 → 取 effect_id → 查 `SYSTEM_EFFECT_ENHANCE_DATA` 得 system_effect_id 列表。
2. 每个 system_effect_id 构造 EffectInfo（含 cur_level/max_level，`min(level, max_level)`）。
3. 再叠加模块效果（`BLUEPRINT_MODULE_EFFECT`）。

> `extend_effect_list` 内的 `return None` 是 pycdc 缩进瑕疵，实际应在 for 循环内继续。

## 九、数据流对比

### 客户端
```
母舰 get_ship_dps
  → get_aircraft_dps(all_effects_list, cur_enhance_dic)
    → _filter_drone_enhance(cur_enhance_dic)  ← 剥除触达载机的强化
    → 子载机.set_mother_ship_info(system_id, filtered_dic)
    → 子载机.prepare_data() → get_effect_list(filtered_dic)  ← 重建子载机 effectList
    → 子载机.get_ship_dps(dps_type)  ← 用重建的 effectList 算火力
```
