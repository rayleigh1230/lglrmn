# 舰队组建 UI 设计

> 状态：设计完成，待实现。数据层已就绪（见 docs/25），本文档描述 UI 层的设计方案。

## 一、设计目标

在现有蓝图系统（ship-list → blueprint-design → enhance）基础上，新增"舰队组建"功能页，让玩家：
- 组建多支编队（旗舰 + 成员）
- 从存档已配好的船（ShipRecord）池中选船加入编队
- 实时查看编队总览（总火力 / 总结构 / 指挥值进度条 / 服役数校验）
- 装载 B 类载机（战机/护航艇）到母舰机库
- 点单船跳蓝图设计页调整配置

## 二、入口与导航

**保持现有栈式导航**（无底部 tabBar），从 ship-list 或 loadouts 进入：

```
loadouts ──(选存档)──► ship-list ──(点船配蓝图)──► blueprint-design ──► enhance
                          │
                          └──(舰队组建入口)──► fleet-build ──(点成员船)──► blueprint-design
```

- 新页面注册：`app.config.ts` 的 pages 数组加 `'pages/fleet-build/index'`
- 跳转方式：`Taro.navigateTo({ url: '/pages/fleet-build/index' })`
- 入口位置：ship-list 页顶部或底部加"舰队组建"按钮（具体位置实现时定）

## 三、核心交互流程

### 主结构：总览区 + 成员列表

```
┌────────────────────────────┐
│ ←  我的编队A         保存  │  顶部固定栏
├────────────────────────────┤
│ 对舰 12500  结构 850K      │  总览区
│ 指挥值 ████████░░ 320/420  │  （火力/结构/指挥值进度条）
│ 服役数 CV3000 3/5  ✓       │  （服役数校验提示）
├────────────────────────────┤
│ ★ 太阳鲸          指挥 55 │  成员列表
│   对舰 4200  载机 5架   >  │  （★=旗舰标记，>=跳蓝图页）
│                            │
│   CV3000          指挥 40 │
│   对舰 6500  载机 5架   >  │
│                            │
│   斗牛级          指挥  6 │
│   对舰 1800             >  │
├────────────────────────────┤
│       ＋ 添加舰船           │  底部固定栏
└────────────────────────────┘
```

### 操作流程

1. **添加舰船**：点"＋添加舰船" → 弹出选船浮窗 → 从存档 ships 池选已配好的 ShipRecord → 选船型 + 数量 → 系统创建对应数量的实例（每个独立 uid）→ 校验指挥值/服役数
2. **设旗舰**：长按或左滑成员卡片 → "设为旗舰"（旗舰标记 ★，用于战斗模拟的集火判定）
3. **调配置**：点成员卡片 → 跳蓝图设计页（`navigateTo /pages/blueprint-design?bpId=...`）→ 改完返回编队页，面板自动重算
4. **移除**：左滑成员卡片 → "移除"
5. **多编队切换**：顶部编队名旁有切换入口（下拉/tab），可新建/重命名/删除编队

## 四、数据来源

### 配置来源：从存档选已配好的船

编队消费 `loadoutStore.active.ships`（已保存的 ShipRecord 池）。每艘船在蓝图设计页配好强化/巅峰/装配后保存为 ShipRecord，编队页从这里选。

- **不是**编队页内嵌蓝图配置（避免页面过重）
- 点编队里的船可跳蓝图页调参，返回后 enhanceStore 的 version 变化触发重算

### 数据流

```
loadoutStore.active.teams[i]  (TeamConfig: flagshipUid + memberUids[])
         +
loadoutStore.active.ships     (Record<uid, ShipRecord>)
         │
         ▼
resolveFormation(store, team, ships, capacityCap, { guildDock })
         │  内部：resolveMemberPanel 逐船算面板（含强化/巅峰/载机 DPS）
         ▼
ResolvedFormation { members, totalFirepower, totalStructure, capacity, valid }
         │
         ▼
编队页 UI 渲染
```

## 五、编队约束（已实现的数据层）

### 指挥值约束

- 舰队总指挥值 = Σ 各船 `getShipCapacity`（取 `cfg_ship[7]`，值域 1-55）
- 上限默认 420（可调 slider，300-420 范围，对应指挥中心等级）
- 超限时进度条变红，`validateFormation` 报"指挥值超限"

### 服役数约束

- 每种蓝图（子型号 bpId）有服役数上限，读 `cfg_ship_blueprint[bpId][7]`（MAX_NUM）
- 档位：护卫/驱逐/战机/护航艇=10（未央15/雷火8）、巡洋=8（未央12）、战巡=6（特殊3）、支援=2-3、航母=5（特殊3）、战列=3
- **公会船坞开关**：开启时跳过服役数校验（可用公会船坞的船突破单蓝图上限）
- 特殊船（雷火之星3/未央15）直接硬编码在字段，无需派系判断

### 战机规格校验（AIRCRAFT_TYPE）

- 战机有规格分类：LIGHT(1) / MID(2) / HEAVY(3)，读 `cfg_ship[19]`
- 机库也分对应规格
- 匹配规则（`isAircraftAvailableForShip`）：HEAVY 机库收全部、MID 收 MID+LIGHT、LIGHT 只收 LIGHT
- 护航艇规格=0（无限制，所有坞舱都能进）

## 六、载机装配

### 两类载机

| 类型 | 来源 | 有独立蓝图？ | 数据层支持 |
|------|------|------------|-----------|
| A类（无人机）| 绑定母舰机库模块（EID=2022/2023）| 无 | `aircrafts: {shipId: number[]}` 模块路径，已支持 |
| B类（战机/护航艇）| 玩家选装机库空位（EID=2020/2021）| 有（83+69条）| `aircraftUids: {shipId: string[]}` 引用 ships 池实例 |

### B类载机装载交互

- 母舰机库有空位（EID=2020 战机机库/EID=2021 护航艇坞舱）→ 玩家选载机装入
- 载机独立存 ships 池（和普通船同构的 ShipRecord，含自身强化/巅峰）
- 母舰 `aircraftUids` 字段存 `{shipId: [uid, ...]}` 引用载机实例
- 装载时两个蓝图（母舰+载机）产生相互作用：载机火力受母舰强化影响（缺陷3 的 `filterDroneEnhance` 透传）
- UI：编队页母舰卡片展开机库区 → 显示空位数 + 已装载载机 → 点空位选载机（规格校验）

### 载机配置存储

载机独立存 ships 池（忠实于客户端逻辑：载机有独立 uid），不是嵌套在母舰 aircrafts 里：
```
ships["u_carrier"] = { uid, shipId:"80201", aircraftUids: {"11701": ["u_ac1","u_ac2"]} }
ships["u_ac1"] = { uid:"u_ac1", shipId:"11701", peakLevel:5, enhanceLevels:{...} }
```

## 七、多编队管理

- 一个存档支持多支编队（`loadoutStore.active.teams: TeamConfig[]`）
- TeamConfig 已实现：`addTeam` / `updateTeam` / `removeTeam`（loadoutStore.tsx）
- UI：编队名旁切换入口，可新建/重命名/删除
- docs/22 提到双编队对抗场景（战斗模拟阶段用），本轮只做管理

## 八、视觉风格

沿用现有深色科幻主题（与蓝图设计页一致）：
- 背景 `#0a0e1a`，卡片渐变 `linear-gradient(135deg, #131a2e, #0f1424)`
- 强调色：青蓝 `#4fc3f7`（激活态）、金色 `#ffc857`（旗舰标记/指挥值）
- CSS 类名前缀 `fb-`（fleet-build），BEM 命名
- 顶栏 + 滚动区 + 底部固定栏的三段式布局（和蓝图页同构）
- 成员卡片：舰船缩略图（`shipThumbnailIcon`）+ 名 + 火力 + 指挥值 + 旗舰标记
- 复用组件：`ShipThumb`（头像）、`StatBar`（属性条）
