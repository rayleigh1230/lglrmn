# 编队数据层总结

> 本文档总结编队数据层的完整实现状态，供后续 UI 开发和战斗模拟参考。

## 一、架构概览

```
fleetFormation.ts（编队数据层核心）
  ├── getShipCapacity        单船指挥值（cfg_ship[7] + 模块加成 EID=2034）
  ├── getTeamCapacity        编队总指挥值（逐船求和）
  ├── getServiceLimit        服役数上限（cfg_ship_blueprint[bpId][7]）
  ├── countByBlueprint       按蓝图 bpId 统计编队成员数
  ├── getAircraftType        战机规格（cfg_ship[19]，LIGHT/MID/HEAVY）
  ├── isAircraftAvailableForShip  规格匹配（哪架战机能进哪个机库）
  ├── validateFormation      编队校验（指挥值 + 服役数 + 成员/旗舰存在性）
  └── resolveFormation       主入口：聚合每船面板 + 指挥值，产出 ResolvedFormation

blueprintCalc.ts（载机 DPS 递归）
  ├── computeAircraftDps     载机 DPS（双轨：A类模块路径 / B类玩家覆写）
  ├── filterDroneEnhance     母舰强化过滤透传给载机（缺陷3，防套娃）
  ├── collectDroneEffectsFromShip  从机库模块收集载机清单
  └── DRONE_EFFECT_IDS       {2020,2021,2022,2023}（排除2024）

techString.ts
  └── levelsToTechStr        强化等级映射 → 科技串（从 editor 下沉到 engine）
```

## 二、数据结构

### ShipRecordInput（fleetFormation.ts）

```typescript
interface ShipRecordInput {
  uid: string;
  shipId: string;
  bpId?: string;
  peakLevel?: number;
  enhanceLevels?: Record<string, number>;
  enabledSlots?: string[];
  aircrafts?: Record<string, number[]>;     // A类载机（无人机）：shipId → 槽数量
  aircraftUids?: Record<string, string[]>;  // B类载机（战机/护航艇）：shipId → uid列表
}
```

### TeamConfigInput

```typescript
interface TeamConfigInput {
  id: string;
  name?: string;
  flagshipUid: string;    // 旗舰 uid（也应在 memberUids 里才会被聚合）
  memberUids: string[];   // 成员 uid 列表
  formation?: { [k: string]: unknown };  // 站位（预留，未实现）
}
```

### ResolvedFormation（resolveFormation 输出）

```typescript
interface ResolvedFormation {
  teamId: string;
  flagshipUid: string;
  members: FormationMember[];  // 每船明细（含完整 BlueprintPanel）
  totalFirepower: { antiShip: number; antiAir: number; siege: number };
  totalStructure: number;
  capacity: FormationCapacity;  // { used, cap, overflow, overflowBy }
  valid: boolean;
}
```

## 四、指挥值校验

- 单船指挥值 = `cfg_ship[7]` + 模块加成（EID=2034 × install_num）
- 舰队总指挥值 = 逐船求和（`getTeamCapacity`）
- 上限由调用方传入（300-420，对应指挥中心等级，当前用默认 420 + 可调）
- 超限时 `validateFormation` 报错"指挥值超限: 用量 X > 上限 Y"

**修正记录**：交接提交 7ad524d 曾误用 [13]（EXPLOIT_CAPACITY，航母 15 万）当指挥值，已回滚到 [7]。

## 五、服役数校验

- 读 `cfg_ship_blueprint[bpId][7]`（MAX_NUM）
- 按 bpId 计数编队成员，超限报错"服役数超限: bpId 用量 X > 上限 Y"
- **公会船坞开关**（`guildDock?: boolean`）：开启时跳过服役数校验
- 特殊船（雷火之星3/未央系15/雷火战机8）直接从字段读，无需派系判断逻辑

## 六、战机规格校验（AIRCRAFT_TYPE）

- 战机规格读 `cfg_ship[19]`：LIGHT=1 / MID=2 / HEAVY=3
- 护航艇规格=0（无限制）
- 规格匹配（`isAircraftAvailableForShip`，对齐客户端 `is_aircraft_avaliable_for_ship`）：
  - HEAVY 机库(3) → 收 HEAVY + MID + LIGHT
  - MID 机库(2) → 收 MID + LIGHT
  - LIGHT 机库(1) → 只收 LIGHT
- AircraftType 枚举值通过 frida hook 确认：`CfgShipField.AircraftType.AIRCRAFT_TYPE_HEAVY=3/MID=2/LIGHT=1`

**dump 过程**：通过 frida attach 游戏进程（PID 6848），直接读 `Tb_cfg_ship.get_all_data()` 返回的 tuple，确认 `get_all_data()` 返回 `{ship_id: tuple}`（tuple 而非 CfgRecordWrap），[19] 即 aircraft_type。156 架战机/护航艇实测验证全部正确。

## 七、载机系统

### 两类载机

| 类型 | EID | 来源 | 有蓝图 | 配置存储 |
|------|-----|------|--------|---------|
| A类（无人机）| 2022/2023 | 母舰机库模块固有 | 无 | `aircrafts: {shipId: number[]}` |
| B类（战机/护航艇）| 2020/2021 | 玩家装入机库空位 | 有（83+69条）| `aircraftUids: {shipId: uid[]}` 引用 ships 池 |

### B类载机强化支持

- 载机独立存 ships 池（和普通船同构的 ShipRecord，含 peakLevel/enhanceLevels/enabledSlots）
- `computeAircraftDps` 新增 `aircraftRecords` 参数：有载机实例时走完整面板解析（`resolveBlueprint` + `resolveBlueprintPanel`，含载机自身强化）；无实例时走白板路径（A类无人机）
- 母舰强化透传给载机：`filterDroneEnhance` 剥除触达 DRONE_EFFECT_IDS 的强化（防套娃），其余透传

### DRONE_EFFECT_IDS

```typescript
new Set([2020, 2021, 2022, 2023])  // 排除 2024（机库容量限制标记）
```

- 2020 EFFECT_CARRIER（战机机库，PARAM = class*100+count）
- 2021 EFFECT_CARRIER_BOAT（护航艇坞舱，PARAM = 纯count）
- 2022 EFFECT_DRONE（无人机，PARAM = ship_id*100+count）
- 2023 EFFECT_ACCOMPANY_DRONE（伴飞无人艇）

## 八、resolveFormation 缺口修复

**问题**：`resolveFormation` 原来调 `resolveBlueprintPanel(store, shipId, '', null, ...)`，第 4 参 null 丢掉了 peakLevel/enhanceLevels，编队面板不反映强化。

**修复**：新增 `resolveMemberPanel(store, rec, ships)`，内部把 `enhanceLevels → levelsToTechStr → resolveBlueprint`（含强化/巅峰）→ `resolveBlueprintPanel`。同时从 `rec.aircraftUids` 收集载机 uid → 查 ships 池 → 传给 resolveBlueprintPanel 的 `aircraftRecords` 参数。

`levelsToTechStr` 从 editor codec 下沉到 engine `techString.ts`（纯逻辑，只依赖 `store.systemEnhance`）。
