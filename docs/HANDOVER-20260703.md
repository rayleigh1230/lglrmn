## 二、关键数据发现

### 抵抗值和护盾值的真实来源 ★
**不在** cfg_ship_type（之前用的[8]是错的），**在** `cfg_module_effect`：

```
cfg_ship_slot (cat=0 装甲模块槽)
  → moduleId (CV3000装甲 = 39995)
    → cfg_module_effect["3999501"] = {EFFECT_ID:10033, EFFECT_PARAM:120}  ← 抵抗值
    → cfg_module_effect["3999502"] = {EFFECT_ID:10021, EFFECT_PARAM:15}   ← 护盾值%
```

验证锚点：CV3000 抵抗=120 ✓ 护盾=15% ✓

### 火力计算链路（已定位，未完全突破）
- 计算器类：`data.ship_attr_calc.BlueprintAttrCalc`
- 入口：`calc.get_ship_dps('ship_firepower'/'ship_aa_firepower'/'ship_siege')`
- 内部用 `eval(attr_type)` 机制取属性，直接调用参数难凑
- DPH 来源已确认：`cfg_weapon_action[key][2]`（混沌场 1200/20 精确验证）
- 当前 UI 用近似公式（每分钟伤害），与游戏实际值有偏差，待后续校准
- 详见 `docs/12-蓝图火力计算研究.md`

### ★ 火力公式完全突破（frida 实战 dump）
- **方法**：frida 注入 `data.ship_attr_calc.BlueprintAttrCalc`，`prepare_by_ship_id` 创建计算器，dump `get_weapon_ship_dps_calc` 表达式树 AST（LazyCalculatable）
- **精确公式**（6武器16锚点零误差）：
  - 周期 period = fireDuration + cooldown（对舰/攻城）
  - cycleFactor = attackRounds(action[0]) × attackCount(action[3]) × 60 × installNum(slot[1]) / period
  - 对舰 = max(dph-10, dph×0.1) × cycleFactor
  - 攻城 = (dph × DESTROY_COEF/100) × cycleFactor
  - 防空 = (base + EID=12300) × airCycleFactor /100 × antiaircraftRatio × 0.15
  - ★防空专用period = dur×(1-EID=12311/100) + cd×(1-EID=12306/100)（防空高效打击+高速循环）
- **校验**：FG300(1543/1851/324) + ST59三武器 + 澄海级两武器(1440/375) 全部精确匹配
- 防空三件套(EID=12300/12306/12311)从 module_effect 取，是舰船特有静态词条
- 详见 `docs/12-蓝图火力计算研究.md`
