## 一、完成功能总览

### 1. 强化项页面（`pages/enhance/`）
竖屏三段式布局，从蓝图页右下角"蓝图强化"FAB 跳入：
- **区域① 科技树**：CSS grid 网格布局，节点按 (BFS列, ui_level行) 对齐，朴素连线，分类标签（预警/伤害/频率/命中），支持鼠标拖曳横向滚动
- **区域② 调校区**：三个六角形（解锁型×2 + 调校型×N），前置链从左到右解锁
- **区域③ 系统导航**：横滚系统图标条，固定底部，点击切换①②内容

### 2. 加点浮窗（`EnhanceSheet`）
- 三模式：multi（双按钮全部强化+强化）、single（单按钮）、choice（二选一选项卡）
- 等级显示 `当前+1/最大`（当前白、+1金，满级无+1）
- 描述显示当前值+增量（原位标注，多数值不错位）
- 满级预览开关（拨开后等级 +N=加满差值，说明显示满级差值）
- 占位符 `{101}{P}{T}{C}` 全量渲染（frida dump 17653 项描述表）

### 3. 调校系统（`TuneSheet` + `TuneRow`）
- 三槽：槽1/2 解锁型（optIdx 11/12，按钮解锁，无等级）→ 槽3 调校型（optIdx 31-43，10级加点面板）
- 前置链：从左到右，前一个未激活则后一个锁定
- 调校浮窗：10圆点等级 + 成功率 + 数值预览

### 4. 强化值联动面板属性
- 全局 Context（`enhanceStore.tsx`）跨页共享强化等级
- `getShipPanel` 通过 `levelsToTechStr` 转 techStr 传 resolver（引擎零改动）
- 强化页点的等级 → 回蓝图页 → 面板结构/火力/抵抗等变化

### 5. 巅峰等级奖励
- 按字段1（`cfg_ship_peak_level`）解析 optIdx70 等级
- per-enhanceItem 精确匹配（`ADJUST_ENHANCE_INDEX`）
- 有扩展的节点显示 ADV 图标（`icon_adv_prop_l.png`）
- maxLevel 真提升（slot.maxLevel + extraMaxLevel）

## 四、逆向突破（frida）

### 4.1 科技树布局规则
- **列号（tree_index）= BFS 跳数**（从根节点出发的广度优先层级）
- **行号（ui_level）= 配置表的 treeColumn 字段**（不是列号！）
- **行序 = LEVEL_ENHANCE_HIGHT 降序**：`{3:0, 0:1, 2:2, 4:4}`（cocos Y向上）
- 来源：dump `SystemEnhanceTree.tree_index_enhance_node_object` + `LEVEL_ENHANCE_HIGHT` 常量

### 4.2 占位符渲染
- 客户端函数 `get_effect_desc(prefix, maxlv, lv)` 渲染描述
- `{101}` = 数值占位符，`{P}/{T}/{C}` = 时间参数
- 数值来源：`get_desc_attr_value_ex(prefix, key, level)` 查 `Tb_cfg_system_skill`
- 批量 dump：读离线 json 拿 (eid, prefix, maxlv) → frida 调 get_effect_desc → 存 enhance_desc_rendered.json

### 4.3 伤害类强化数值
- 3774 个强化项（21%）EFFECT_PARAM/EFFECT_ID 为空（伤害类）
- 数值来源：`get_cur_and_next_level_value(prefix, key, level)` → 返回 (当前值, 增量)
- 批量 dump → enhance_values.json（value + effect_id）

### 4.4 武器伤害类型
- **weapon_action[1]**：1=能量(125条), 2=实弹(403条), 4-9=非攻击武器(55条)
- cfg_weapon 表无伤害类型字段（全 20 字段穷举确认）
- 之前误用 action[3]（14053脉冲炮和18021龙式火炮 action[3] 都=3 但伤害不同）

### 4.5 巅峰强化扩展
- per-enhanceItem（不是 per-slot）：optIdx70 的 `ADJUST_ENHANCE_INDEX` 指向被扩展的普通 optIdx
- 解锁等级从 `cfg_ship_peak_level` 字段1解析（`slotId70,level` 格式）

### 4.6 二选一数据模式
- 同 `(slotId, treeColumn)` 桶内 ≥2 个 `nodeFlag≠0` 节点互斥（111 处）
- 未选时合并为单个图标（flag 最小的主选项位置）

### 4.7 待逆向（已知缺口）
- **弹道类型（直射/投射）**：不在 weapon_action 5 个元素里，在加密字节码判定。只影响战斗结算前摇，不影响面板。
- **强化项描述占位符精确映射表**：当前用 frida dump 的渲染表兜底，逆向出 `{101}`→数值的映射规则后可脱离 dump 依赖。
