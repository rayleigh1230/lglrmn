### B. 状态视觉统一（用户多次反馈修正）

统一三态规则：**边框高亮=已解锁，图标高亮=已投入**

| 状态 | 强化项（方框） | 调校项（六角形） |
|------|---------------|-----------------|
| locked | 边框暗 + 图标灰 | 灰图标（不用锁图标） |
| available | **边框明亮蓝** + 图标灰 | 底图蓝色描边发光 + 图标灰 |
| acquired/active | 边框蓝 + **图标高亮** | 底图正常 + 图标高亮 |

关键修复：`opacity/grayscale` 下沉到 `.et-icon`/`.tr-hex-icon` 子元素，避免压暗边框。

### C. 调校系统数据修正（2个重要 BUG）

#### BUG1：前置解锁槽识别错误
- **根因**：`resolveTuneRowVM` 用 `optIdx 11-13 + UNLOCK_TYPE=2` 识别前置解锁槽，但实际 optIdx 在 4-12 浮动
- **修复**：改为仅 `UNLOCK_TYPE === 2`（不限 optIdx）
- **数据验证**：全量 4343 条 UNLOCK_TYPE=2 记录，全部有 UNLOCK_COST_RARITY + cost=[0]，无误伤
- **文档查证**：docs/13-巅峰等级与调校系统机制.md §2.1 三步链确认

#### BUG2：调校槽识别错误（白垩级图标不显示）
- **根因**：`resolveTuneSystem`（引擎层）用 `optIdx 31-43` 识别调校槽，但少量调校槽 optIdx 在 7/12/50/51（如白垩级 521010112 optIdx=12）
- **修复**：改为仅 `ADJUST_PROB` 存在（不限 optIdx）
- **数据验证**：全量 2254 条调校槽，98% 在 31-43，2%（23条）在范围外被错误排除

### D. 孔位上限强制约束（BUG 修复）

- **问题**：孔位数（ENHANCEMENTS_LIMIT）只显示没强制，孔位满后仍能给新强化项加点
- **修复**：`EnhanceSheetVM` 加 `slotsFull` 字段，按钮区加「强化次数已满」分支
- **判定**：`cur === 0 && usedSlots >= enhanceLimit`（已加点项不再受限）

### E. 强化页布局调整

1. **系统排缩小**：图标 84→56，项宽 130→96，整体高度 150→90px
2. **底部功能区**（页面最底部）：重置（可用）+ 保存/导入/分享（占位）
   - 重置：`Taro.showModal` 二次确认，清空当前 slotId 下所有强化+调校+解锁
   - 保存/导入/分享：占位按钮，提示「功能开发中」

### F. 强化页状态架构修复

- **BUG**：`Maximum update depth exceeded`（无限循环）
- **根因**：`updateAcquired` 同步写全局 store 触发 version bump → useEffect 监听 version 又写本地 → 循环
- **修复**：`useRef` 标记本页发起的更新，useEffect 跳过自己触发的 version 变化

### G. 超主力舰装配选择联动

- **问题**：强化页系统排不反映蓝图页的切换组选择（M1/M2）
- **修复**：蓝图页跳转时序列化 `enabledSlots` 进 URL；强化页解析后传给 `resolveEnhanceSystem`/`resolveSystemNav`/`resolveEnhanceTreeVM`
- **过滤**：切换组未选中成员（isActive=false）不显示

### H. 蓝图页系统排孔位联动

- **改动**：蓝图页系统排孔位从纯空方块改为按强化等级着色（与强化页导航条联动）
- **共用函数**：`computeSlotStates`（enhanceView.ts 导出，两边调用）

## 二、技术决策记录

### 孔位计数语义
`ENHANCEMENTS_LIMIT`（如5）与实际强化项数（可能6-7）无固定映射。采用**按计数**：
- 满级强化项数 → 白点数
- 已投入未满级数 → 灰点数
- 剩余 → 空圈

### 调校项图标联动
调校项图标用其 target 强化项的 `effectPrefix` 解析（原版二合一图标语义），实现关联强化槽和调校槽用一样的图标。
